AI Model Gateway · SaaS Platform

大模型调用中转平台

小乐AI 是一套面向平台运营、代理商和终端用户的多模型中转系统。项目以 FastAPI 后端、Vue2 管理前台和 MySQL 数据库为核心，统一管理 OpenAI、Anthropic、Responses 和图片模型调用，提供渠道容灾、计费、套餐、兑换码、代理白标站点与全链路审计能力。

System Architecture

从前台门户到多渠道转发的完整链路

系统通过统一 API 域名、站点上下文识别和模型路由层，将平台直营与代理白标站点放在同一套代码与数据库中运营。

01 Frontend

Vue2 单页应用

登录、用户中心、管理后台、代理门户共享一套前端构建产物。

02 Gateway

Nginx / API 域名

生产环境前端静态部署，API 反代到 FastAPI 服务。

03 Backend

FastAPI 服务层

认证、路由、计费、日志、套餐和代理资产管理。

04 Data

MySQL 业务数据

用户、API Key、渠道、统一模型、日志、余额、套餐与代理表。

05 Upstream

多模型上游

对接 Claude、GPT、Gemini、Grok、Codex 与图片生成/编辑通道。

Role Portals

三类角色，一套多租户运营体系

路由守卫根据用户角色区分平台管理员、代理商和普通用户，后端通过站点上下文与 agent_id 做数据隔离。

/admin

平台管理员

  • 管理渠道、统一模型、模型映射和覆盖规则。
  • 管理用户余额、图片积分、套餐、兑换码与系统配置。
  • 查看请求日志、消费流水、健康监控和数据库连接池状态。
  • 创建代理站点并给代理分配余额、图片积分和套餐库存。
/agent

代理商门户

  • 拥有独立前台域名、站点文案、公告和联系方式。
  • 管理自身范围内的用户、充值、扣减、兑换码和套餐发放。
  • 查看本代理下的请求记录、消费流水和使用排行。
  • 通过统一 API 域名服务代理用户，同时保留白标体验。
/user

终端用户

  • 登录注册后创建、启用、禁用和查看 API Key。
  • 使用 AI 对话页面、移动端对话和模型列表。
  • 查看余额、图片积分、套餐状态、消费明细与用量统计。
  • 通过兑换码充值，并按 OpenAI 或 Anthropic 客户端方式接入。

API Integration

兼容主流 AI 客户端与 SDK

平台入口同时支持带 /v1 和不带 /v1 的路径，方便 OpenClaw、Cursor、Continue、Cline、OpenAI SDK 和 Anthropic SDK 接入。

协议 / 能力 标准路径 兼容路径 说明
OpenAI Chat Completions POST /v1/chat/completions POST /chat/completions 面向 OpenAI SDK、Cursor、Continue、Cline 等客户端。
OpenAI Models GET /v1/models GET /models 返回平台暴露给用户的可用统一模型列表。
OpenAI Responses POST /v1/responses POST /responses 兼容 Responses API 与部分 Codex CLI 使用场景。
Anthropic Messages POST /v1/messages POST /messages 支持 anthropic-api-key 与 X-API-Key 鉴权头。
图片生成 / 编辑 POST /v1/images/generations POST /v1/images/edits 使用图片积分计费，支持 OpenAI、Google、Vertex 等图片上游。
站点公开配置 GET /api/public/site-config GET /api/user/profile/site-config 登录页、用户端和代理站点读取当前域名对应的平台或代理配置。

Deployment

本地开发与生产部署路径

部署文档中推荐后端 FastAPI 运行在 8085，前端 Vue 构建为静态文件，由 Nginx 提供访问并反向代理 API。

本地启动

# 后端
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8085 --reload

# 前端
cd frontend
npm install
npm run serve

生产部署

  • Python 3.9+、Node.js 16+、MySQL 8.0+、Nginx 1.18+。
  • 后端通过 `.env` 覆盖数据库、JWT、CORS 和服务端口配置。
  • 前端执行 `npm run build` 后将 `dist` 静态文件交给 Nginx。
  • 代理前台可使用独立域名，统一 API 推荐共享 `api` 域名。
生产安全提示：说明页不展示仓库配置中的数据库连接、JWT Secret 或上游 Key 示例。上线前请确认敏感配置均由环境变量或安全配置系统注入，并避免将真实凭据写入 SQL 初始化脚本或公开文档。

Operations Notes

后续维护重点

这些注意事项来自当前代码与文档的综合分析，适合作为上线前检查和后续重构优先级参考。

上线前安全检查

  • 生产环境必须用 `.env` 或安全配置系统覆盖数据库连接串、JWT Secret、上游 API Key 和 CORS 白名单。
  • 反向代理应明确控制可信 Host 与 API 域名，避免客户端伪造站点上下文造成代理数据边界混淆。
  • API Key 当前支持 reveal 完整密钥，若转为更严格的生产安全模型，可考虑仅创建时展示一次。
  • 管理员、代理商和用户端的充值、扣减、套餐发放属于资产操作，需保留操作日志并限制权限范围。

工程演进建议

  • proxy_service.py 承担了协议适配、计费、缓存、日志、图片和容灾，后续可按职责逐步拆分。
  • 高并发流式代理场景下，需要持续观察同步 SQLAlchemy Session、连接池占用和上游超时行为。
  • 模型、渠道、套餐、图片积分和代理库存均有管理端入口，数据库升级脚本应和页面能力同步维护。
  • 新增客户端兼容能力时，优先补充 docs/client-integration-guide.md 与用户端快速开始页面。
