# CPA缓存计费 Impl

日期：20260506

## 任务概述

本次改造取消系统内部请求体分段缓存处理，不再执行 Redis 分段缓存读写，也不再在目标页面展示 `内部缓存/PARTIAL/读段/建段/复用` 等内部分析结果。

新的缓存依据改为 CPA/上游响应中的真实 usage 字段：

- OpenAI Chat Completions：`usage.prompt_tokens_details.cached_tokens`
- Anthropic Messages：`usage.cache_read_input_tokens`、`usage.cache_creation_input_tokens`

计费口径：

- 普通输入按输入价格计费。
- 输出按输出价格计费。
- 缓存创建只展示，不额外计费。
- 缓存读取按输入价格的 10% 计费。

## 核心代码说明

### 内部缓存停用

`CacheMiddleware.wrap_request` 和 `StreamCacheMiddleware.wrap_stream_request` 改为纯透传，不再调用 `RequestBodyCacheService.analyze_request`。WebSocket Responses 里直接调用内部缓存分析的路径也已改为 `cache_info = None`。

保留旧类和方法签名，是为了兼容现有代理分支调用，不扩大重构范围。

### CPA/OpenAI 缓存解析

新增 `_extract_openai_prompt_cache_summary`：

- `cached_tokens > 0`：记录为缓存读取，普通输入 token 为 `prompt_tokens - cached_tokens`。
- CPA 8317 渠道且 `cached_tokens = 0`：将本次 prompt 记录为缓存创建展示值。
- 非 CPA 渠道不会仅凭 `cached_tokens = 0` 展示缓存创建。

### 计费拆分

`_deduct_balance_and_log` 现在拆分：

- `input_cost`
- `cache_read_cost`
- `output_cost`
- `total_cost = input_cost + cache_read_cost + output_cost`

新增字段保存：

- `billable_input_tokens`
- `billable_cache_read_input_tokens`
- `cache_read_cost`
- `input_price_per_million_snapshot`
- `output_price_per_million_snapshot`
- `price_multiplier_snapshot`
- `token_multiplier_snapshot`

这些字段同步写入 `request_log` 和 `consumption_record`，用于历史账单详情稳定展示。

### 前端展示

目标页面已移除内部缓存展示：

- 不再显示 `内部缓存`
- 不再显示 `PARTIAL/HIT/MISS`
- 不再显示 `读 N 段/建 N 段`
- 不再显示 `复用 ~N tok`

用量区改为展示真实计费输入、输出、缓存创建、缓存读取。缓存读取主展示统一使用 `billable_cache_read_input_tokens`，避免 `token_multiplier` 不为 1 时展示 token 与合计、费用不一致。详情中展示输入、输出、缓存读取成本拆分，缓存创建标记为不额外计费。
