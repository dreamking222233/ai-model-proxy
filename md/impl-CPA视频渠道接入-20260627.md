# impl-CPA视频渠道接入-20260627

## 任务概述

新增 `cpa-grok-video` 渠道类型，用于通过 CPA 调用 Grok 视频生成，同时保持用户侧 `/v1/videos`、`/v1/created/video`、`/v1/videos/{id}`、`/v1/videos/{id}/content` 的返回结构不变。

## 文件变更清单

  - 新增 CPA Grok 视频创建、查询、下载 URL 适配。
  - 将 CPA 原生 `request_id/status/video.url` 标准化为现有视频 JSON。
  - CPA 请求秒数按 1 到 15 夹取，超过 15 按 15 生成和计费。
  - 下载/同步等待计费前，必须先拿到真实视频 URL 并校验内容可访问。

## 核心代码说明

- 渠道仍使用 `protocol_type=openai`，新增 `provider_variant=cpa-grok-video`，避免影响现有 OpenAI/Anthropic/Google 主分支。
- CPA 创建任务内部走 `/v1/videos/generations`，但返回给用户的创建 JSON 仍使用当前系统字段，如 `id`、`object`、`status`、`usage`。
- CPA 查询任务内部走 `/v1/videos/{id}`，将 `done` 转为 `completed`。
- CPA 下载内容时先查询任务状态，只有状态完成且存在 `video.url` 时才校验视频内容并进入扣费逻辑。
- 用户端三个入口统一使用后端模型能力控制视频参数：CPA Grok 为 1-15 秒，老视频渠道未下发能力时继续使用原 6/10/12/16/20 秒。
- 移动端图生视频把上传参考图按原有 `input_reference[]` multipart 字段提交，后端再转 CPA 需要的 data URL，不改变用户调用方式。
- Review 后修复同步等待接口的 CPA 时长计费归一化，避免 15 秒任务完成后被旧视频渠道时长规则误拒绝。
- Review 后修复移动端已缓存 API Key 时不恢复历史视频 blob 映射的问题。
