# 聊天图片缓存与模型筛选优化实施记录（2026-04-23）

## 任务概述
- 修复聊天页图片“查看原图”跳转 `about:blank#blocked`。
- 让聊天页图片结果在刷新后仍能恢复，直到用户删除对应会话或清空全部会话。
- 图片支持点击放大预览，并支持下载原图。
- 编辑图片发送后清空上传区和输入区。
- 优化聊天页模型选择区域，支持按厂商、类型、关键词筛选。

## 文件变更清单
- `frontend/src/utils/chatStorage.js`
- `frontend/src/views/common/AiChat.vue`
- `frontend/src/components/chat/ChatMessage.vue`
- `frontend/src/components/chat/ModelSelector.vue`

## 核心实现说明
### 1. 聊天图片持久缓存
- 在 `chatStorage.js` 中新增 `IndexedDB` 图片缓存能力。
- 会话消息仍只保存图片 `cacheKey`，图片实际内容存入浏览器 `IndexedDB`，避免把大图 base64 直接写进 localStorage。
- 新增能力包括：
  - 保存单张图片缓存
  - 按 `cacheKey` 读取图片
  - 批量回填当前会话图片
  - 删除会话时清理关联图片
  - 清空全部会话时清理关联图片

### 2. 刷新后恢复图片
- `AiChat.vue` 在页面初始化、切换会话、删除会话时，会根据消息中的 `cacheKey` 对运行时图片映射进行回填或清理。
- 生图结果和修图结果在成功返回后，先写入 IndexedDB，再写回会话消息，确保刷新后仍能恢复。

### 3. 原图打开 / 放大 / 下载
- `ChatMessage.vue` 不再直接使用 `data:` URL 配合 `target="_blank"`。
- 改为：
  - 点击“查看原图”时由父组件把缓存图片转换成 `blob:` URL 再新开标签页
  - 点击图片本身时打开预览弹窗
  - 点击“下载原图”时通过 `blob` 下载
- 预览弹窗支持：
  - 查看大图
  - 下载当前预览原图

### 4. 图片编辑发送后的清理
- 编辑图片发送时会先把原图缓存到浏览器，并把引用写入消息。
- 随后立即清空当前上传区和输入内容，避免同一张本地图片残留在输入区域。

### 5. 模型选择区域优化
- 将原来单一的模型下拉框改为弹出式模型选择面板。
- 新面板支持：
  - 关键词搜索
  - 按厂商筛选
  - 按模型类型筛选
  - admin 模式下保留渠道切换
- 交互上更接近 `user/models` 页面，但保持聊天页顶部区域的紧凑布局。
