# 代理套餐时间显示修复 Impl

## 任务概述

修复 `agent/subscription` 套餐发放记录时间多显示 8 小时的问题，并确认 `admin/subscription` 页面当前使用的时间口径。

## 文件变更清单

- `frontend/src/views/agent/SubscriptionManage.vue`
  - 将发放记录表格时间渲染从 `formatUtcDate()` 改为 `formatDate()`。
- `md/plan-代理套餐时间显示-20260504.md`
  - 记录排查与修复方案。

## 核心代码说明

后端 `SubscriptionService.get_current_time()` 返回 `Asia/Shanghai` 的无时区时间，套餐字段 `start_time`、`end_time`、`activated_at` 也是按该北京时间写入。

`agent/subscription` 原先使用 `formatUtcDate()`，会把无时区字符串当 UTC 解析。例如后端返回 `2026-05-04T00:06:37` 时，浏览器本地显示会变成 `2026-05-04 08:06:37`。

现在改为 `formatDate()`，与 `admin/subscription` 保持一致，按本地时间直接显示。

## Admin 页面时间口径

`admin/subscription` 当前使用 `formatDate()`：

- 套餐记录开始/结束时间：`formatDate(record.start_time)` / `formatDate(record.end_time)`
- 套餐记录创建时间：`customRender: text => formatDate(text)`
- 使用详情调用时间：`formatDate(text)`

因此 admin 页面当前展示的是后端返回的本地时间口径，不会把套餐时间额外当 UTC 加 8 小时。
