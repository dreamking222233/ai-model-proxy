## 任务概述

本次实现基于现有代理站点架构，为代理前台新增“自定义域名”支持能力，同时保持后端 API 继续统一使用 `https://api.xiaoleai.team`。

本轮按第一阶段最小可行版落地：

- 沿用 `agent.frontend_domain` 作为代理唯一生效前台域名
- 支持录入任意合法前台域名，例如 `test.test.com`
- 新增后端动态 CORS 白名单，允许代理自定义域名跨域访问共享 API
- 管理端新增代理域名填写说明与前端校验
- 补充部署文档与上线核对清单

## 核心实现说明

### 1. 动态 CORS 白名单

新增 `backend/app/core/cors.py`：

- 静态继续兼容平台域名、本地域名、`*.xiaoleai.team`
- 动态从数据库读取 `agent.frontend_domain`
- 对 `OPTIONS` 预检请求直接返回允许头
- 对正式请求自动补充 `Access-Control-Allow-Origin`
- 增加短缓存，并在代理创建/更新成功后主动刷新缓存

### 2. 代理域名输入规范化

在 `AgentService` 中新增代理域名输入规范化与校验逻辑：

- 支持自动去掉 `http/https`
- 自动去端口并转小写
- 拒绝路径、查询参数、片段、空格
- 禁止把平台前台域名和平台 API 域名配置成代理域名
- 保留本地开发域名与 IP 的兼容

解决了旧逻辑下把 `https://test.test.com` 误解析成 `https` 的问题。

### 3. 管理端输入提示与前端校验

在代理管理页中：

- 增加“纯域名填写”说明
- 保存前先做前端规范化
- 对明显错误输入直接弹提示，不再把脏值发到后端

### 4. 数据库影响

本轮无数据库结构变更。

原因：

- 现有 `agent.frontend_domain` 字段和唯一索引足以支撑“一个代理一个前台域名”
- 本轮不引入多域名表

仅需更新代理配置数据，不需要执行升级 SQL。
