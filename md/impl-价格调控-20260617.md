# 价格调控实施记录

## 任务概述

本次实现按模型系列、模型类型、计费类型进行价格调控：

- 在 `admin/models` 增加模型系列归类字段，支持 `gpt`、`claude`、`grok`、`gemini`、`other`。
- 新增后台 `admin/price-adjustments` 价格调控页面。
- 支持长期规则和每日北京时间时间段规则，包含跨天时间段，例如 `23:00-07:00`。
- 文本计费使用 `全局价格倍率 x 分类调控倍率 x Fast倍率 x 长上下文倍率`。
- 图片/视频媒体积分计费使用 `基础媒体积分 x 分类调控倍率`，不叠加全局 USD 价格倍率。
- 修复 `user/asset-source` 页面时间显示为北京时间，并复核推广相关页面。

## 核心代码说明

### 模型归类

`ModelService.infer_model_series()` 按模型名前缀推断系列：

- `gpt`、`o1`、`o3`、`o4` => `gpt`
- `claude` => `claude`
- `grok` => `grok`
- `gemini` => `gemini`
- 其他 => `other`

管理员可在 `admin/models` 手动覆盖归类。

### 规则匹配

`PriceAdjustmentService.resolve_rule()` 按以下条件匹配：

- `enabled = 1`
- `model_series` 精确匹配或 `all`
- `model_type` 精确匹配或 `all`
- `billing_type` 精确匹配或 `all`
- `schedule_type=always` 直接生效
- `schedule_type=daily_time` 使用北京时间判断

多条规则同时命中时，按 `priority ASC, id DESC` 取第一条。每日时间窗支持跨天：`start_time > end_time` 时按 `now >= start_time OR now < end_time` 判断。

### 文本计费

文本预检和实际扣费均调用 `PriceAdjustmentService.resolve_multiplier()`，公式为：

```text
base_price * (system_config.price_multiplier * adjustment_multiplier) * fast_multiplier * context_multiplier
```

为兼容已有日志字段，`price_multiplier_snapshot` 记录 `system_config.price_multiplier * adjustment_multiplier`。

### 媒体积分计费

图片生成、图片编辑、视频生成的媒体积分扣费公式为：

```text
base_media_credits * adjustment_multiplier
```

媒体积分不叠加 `system_config.price_multiplier`，避免全局美元价格倍率影响积分体系。

### 北京时间

`PriceAdjustmentService` 使用 `ZoneInfo("Asia/Shanghai")` 解析规则当前生效状态。前端 `user/asset-source` 改用 `formatBeijingTime`。已复核：

- `frontend/src/views/admin/PromotionManage.vue` 已使用 `formatBeijingTime`。
- `frontend/src/views/user/Promotion.vue` 已使用 `formatBeijingTime`。
- `frontend/src/views/agent/PromotionManage.vue` 已使用 `formatBeijingTime`。

## 接口文档

管理端价格调控接口前缀：`/api/admin/price-adjustments`

- `GET /api/admin/price-adjustments`：分页查询规则，参数 `page`、`page_size`、`model_series`、`model_type`、`enabled`。
- `GET /api/admin/price-adjustments/options`：返回系列、模型类型、计费类型、生效方式选项。
- `GET /api/admin/price-adjustments/effective`：返回当前各系列/类型有效倍率概览。
- `POST /api/admin/price-adjustments`：新增规则。
- `PUT /api/admin/price-adjustments/{rule_id}`：更新规则。
- `DELETE /api/admin/price-adjustments/{rule_id}`：删除规则。
