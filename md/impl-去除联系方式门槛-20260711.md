# 去除联系方式门槛实施记录

## 任务概述

移除平台直营客户 QQ、微信联系方式的充值金额门槛，恢复直接展示；代理端配置和展示逻辑保持不变。

## 文件变更清单

- `backend/app/api/user/profile.py`：删除支付订单累计金额计算、联系方式脱敏和相关响应字段。
- `backend/app/api/public/site.py`：恢复平台公开站点配置直接返回 QQ、微信。
- `backend/app/api/admin/system.py`：移除固定公告配置的门槛字段与保存逻辑。
- `frontend/src/views/admin/AnnouncementManage.vue`：移除门槛输入与说明。
- `frontend/src/views/user/Dashboard.vue`、`frontend/src/views/user/Recharge.vue`、`frontend/src/layout/UserLayout.vue`：移除锁定提示，直接展示配置的联系方式。
- `backend/sql/initData.sql`、`sql/initData.sql`：移除门槛默认配置。
- `backend/sql/upgrade_remove_support_contact_threshold_20260711.sql`：删除已有数据库中的旧门槛配置。

## 核心实现

用户站点配置、固定开屏公告和公开站点配置都直接使用 `AgentService.build_public_site_config` 返回的联系方式。管理员只维护公告标题、内容、微信和 QQ；不再存在用户付费金额查询或门槛配置。为兼容近期前端和调用方，原 `support_contact_*` 响应字段保留为废弃字段，固定返回联系方式可见、无提示、无门槛。

数据库升级 SQL 使用按键删除，重复执行不会产生副作用。

## 测试验证

- `python3 -m py_compile backend/app/api/user/profile.py backend/app/api/public/site.py backend/app/api/admin/system.py`：通过。
- `npm run build`：通过，仅有现有前端资源体积告警。
- 本地数据库已执行 `backend/sql/upgrade_remove_support_contact_threshold_20260711.sql`，并验证 `platform_support_contact_threshold_cny` 配置记录数量为 `0`。
