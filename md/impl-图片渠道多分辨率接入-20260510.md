## 任务概述

本次完成了图片渠道多分辨率接入改造，目标包括：

- `/admin/models` 可为生图模型配置分辨率规则
- `gpt-image-2` 支持 `1K / 2K / 4K` 三档积分配置
- 新增 OpenAI 图片原生尺寸渠道类型，支持向上游透传 `size / quality`
- 用户请求图片时，可从 `image_size / imageSize / size` 中识别分辨率档位
- 当存在“仅支持默认 1K”的旧渠道和“支持 1K/2K/4K”的新渠道时，请求会按分辨率筛选可用上游，避免错路由和错计费

## 核心实现说明

### 1. 生图模型分辨率配置不再限定为 Google

后端新增 `ModelService.supports_image_resolution_rules(...)`，将“是否允许配置分辨率规则”从：

- `protocol_type == google`

放宽为：

- `model_type == image`
- `billing_type == image_credit`
- 模型已声明尺寸能力

这样 `gpt-image-2` 也能在 `/admin/models` 中配置分辨率档位。

### 2. `/admin/models` 支持 `gpt-image-2` 分辨率配置

前端模型管理页改为对所有“支持尺寸能力的生图积分模型”展示分辨率配置区。

新增 `gpt-image-2` 默认预设：

- `1K = 0.5`
- `2K = 1`
- `4K = 2`

同时补充了管理端可视化信息：

- 模型列表直接展示图片模型的“尺寸能力”
- 模型列表直接展示“是否支持编辑图”
- 映射表展示渠道类型、分辨率能力、是否支持编辑图
- 映射弹窗在选择渠道时展示能力说明，提醒 `1K` 兼容渠道与 `Native Size` 渠道的差异

### 3. 新增 OpenAI 图片原生尺寸渠道类型

渠道层新增：

- `openai-image-native-size`

语义区分：

- `openai-image-compatible`
  - 适合旧上游
  - 仅支持默认 `1K`
  - 继续走 prompt 适配尺寸/比例
- `openai-image-native-size`
  - 适合新上游
  - 支持 `1K / 2K / 4K`
  - 直接向上游透传 `size / quality`

健康检查逻辑已同步识别该新变体。

### 4. 用户尺寸参数识别与档位归一

`ProxyService` 新增了以下能力：

- 识别 `image_size / imageSize / size`
- 识别像素 `size`，例如：
  - `1024x1024`
  - `1024x1792`
  - `2048x2048`
  - `3840x2160`
- 从像素尺寸归一出计费档位：
  - `<= 1792` -> `1K`
  - `<= 3584` -> `2K`
  - 更高 -> `4K`
- 从像素尺寸反推比例：
  - `1:1 / 16:9 / 9:16 / 3:2 / 2:3 / 4:3 / 3:4`

同时增加一致性校验：

- 若同时传 `image_size` 和像素 `size`
- 且两者对应档位不一致
- 直接返回 `INVALID_IMAGE_SIZE`

避免出现“按 1K 计费、按 4K 生成”的错误场景。

### 5. 计费规则与渠道能力彻底解耦

当前分辨率处理拆成两层：

1. 模型层决定本次应该按哪个档位计费
2. 渠道层决定当前上游是否支持该档位

新增渠道能力过滤逻辑后：

- 用户请求 `4K`
- 若某个渠道只有默认 `1K`
- 该渠道会在请求前被过滤掉

这样可以避免：

- 请求 `2K/4K` 时误命中只支持 `1K` 的旧上游
- 按高档位计费却实际只生成默认档位

### 6. OpenAI 原生尺寸渠道透传 `size / quality`

对于 `openai-image-native-size`：

- 文生图透传 JSON 字段：
  - `model`
  - `prompt`
  - `n`
  - `size`
  - `quality`
  - `response_format=b64_json`
- 图片编辑透传 multipart 字段：
  - `image`
  - `model`
  - `prompt`
  - `n`
  - `size`
  - `quality`
  - `response_format=b64_json`

若用户未直接传像素 `size`，系统会根据：

- `image_size`
- `aspect_ratio`

合成默认像素尺寸。

### 7. 上游错误信息中文化

针对图片渠道新增了上游错误文案归一逻辑：

- `stream disconnected before completion`
- `rate limit`
- `timeout`
- `invalid image`
- `content policy`

这类常见英文错误会被翻译成中文后再返回给用户，便于前端直接展示。
