# 套餐-额度24小时刷新 Impl

## 任务概述

修复套餐额度按自然日 0 点刷新的漏洞，改为以用户当前套餐开通时间 `user_subscription.start_time` 为锚点，每 24 小时刷新一次额度。

本次实现覆盖管理端开通、代理端发放、用户端在线购买后的统一扣费链路，因为三类入口最终都复用 `SubscriptionService` 的套餐周期能力。

## 核心代码说明

### rolling 24h 周期

新增 `_get_subscription_cycle_window(subscription, now_local)`：

- 固定按北京时间处理。
- `cycle_start_at = subscription.start_time + N * 24h`
- `cycle_end_at = cycle_start_at + 24h`
- 最后一个周期不超过 `subscription.end_time`
- `now >= end_time` 时使用 `end_time - 1 microsecond` 计算，避免日卡到期点出现下一周期。

### 旧周期兼容

`subscription_usage_cycle` 表结构保持不变，仍使用 `(subscription_id, cycle_date)` 唯一键。

新逻辑中：

- `cycle_date` 表示 rolling 周期开始日期。
- `cycle_start_at/cycle_end_at` 是精确刷新窗口。
- 如果已有周期行的窗口仍是旧自然日 0 点，写路径会调用 `_sync_cycle_snapshot(...)` 重建窗口快照。
- 如果新窗口周期不存在，创建前先调用 `_rebuild_cycle_usage_snapshot(...)` 聚合已有消费记录，防止用户获得额外空额度。

### 用户端展示

`get_current_subscription_summary(...)` 返回：

- `refresh_anchor_at`
- `next_refresh_at`
- `refresh_period_hours`

`user/balance` 页面套餐卡片显示“下次刷新”，格式为 `YYYY-MM-DD HH:mm:ss`。
