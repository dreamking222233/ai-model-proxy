# 套餐列表性能优化 Impl

## 任务概述

修复套餐 24 小时刷新上线后，管理端套餐用户列表和用户 profile 偶发慢响应的问题。

## 文件变更

- `backend/app/services/subscription_service.py`
  - `_get_cycle_for_summary(...)` 新增 `rebuild_from_consumption` 参数。
  - 默认 summary 读路径不再扫描 `ConsumptionRecord` 重建快照。
  - 单条使用详情页保留 `rebuild_from_consumption=True`，仍可获取精确周期用量。

- `backend/test_subscription_plan_quota_edit.py`
  - 增加断言，确认 summary 默认不会调用 `_rebuild_cycle_usage_snapshot(...)`。

- `backend/sql/upgrade_subscription_cycle_performance_20260624.sql`
  - 新增 `idx_consumption_subscription_created_id(subscription_id, created_at, id)`。

- `backend/sql/init.sql`
- `backend/sql/initData.sql`
  - 新环境初始化同步新增索引。

## 核心说明

本次优化不改套餐扣费写路径。请求前额度检查和请求后扣费仍会通过 `_get_or_create_cycle(...)` 精确创建或重建 rolling 24 小时周期，确保日卡不会在 0 点额外刷新。

展示接口改为读取周期表快照，避免每次打开列表或 profile 都按用户历史消费记录聚合大表。
