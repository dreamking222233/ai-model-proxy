# 套餐用户列表 Impl

## 任务概述

在 `admin/subscription` 页面新增“套餐用户”页签，用于查看当前拥有有效套餐的用户，按剩余时长排序，并快速筛选即将到期用户。

- `backend/app/api/admin/subscription.py`
  - 新增 `GET /api/admin/subscription/active-users`。

- `backend/app/services/subscription_service.py`
  - 新增 `list_active_subscription_users`。
  - 仅查询当前有效套餐。
  - 按用户维度去重，同一用户多条当前有效套餐只返回结束时间最晚的一条。

## 核心代码说明

`active-users` 接口只返回满足以下条件的套餐：

- `status` 为 `active/null/""`
- `start_time <= 当前时间`
- `end_time >= 当前时间`

接口按 `user_id` 做用户级展示。同一用户存在多条当前有效套餐记录时，通过窗口排序选择 `end_time` 最晚、`id` 最大的一条，避免“套餐用户”列表和总数重复。

排序规则：

- `sort_order=asc`：按 `end_time ASC`，快到期优先。
- `sort_order=desc`：按 `end_time DESC`，剩余时长最多优先。

到期筛选：

- `expires_within_days=7` 会返回 7 天内到期的有效套餐用户。
