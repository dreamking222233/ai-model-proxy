# 套餐记录查询 Impl

## 任务概述

在管理端 `admin/subscription` 的“套餐记录”页面新增按用户 ID、用户名或邮箱查询套餐记录的能力，便于管理员直接定位某个用户的历史和当前套餐。

## 文件变更清单

- `backend/app/api/admin/subscription.py`
  - `/api/admin/subscription/list` 新增 `keyword` 查询参数。

- `backend/app/services/subscription_service.py`
  - `list_all_subscriptions` 新增 `keyword` 参数。
  - 数字关键字支持用户 ID 精确匹配，同时继续匹配用户名/邮箱。
  - 非数字关键字按用户名、邮箱模糊匹配。

- `frontend/src/views/admin/SubscriptionManage.vue`
  - “套餐记录”工具栏新增搜索框。
  - 支持回车/点击查询、清空搜索、重置筛选。
  - 列表请求携带 `keyword`。

- `backend/app/test/test_subscription_compatibility.py`
  - 新增按用户名查询套餐记录测试。
  - 新增按用户 ID 查询套餐记录测试。
  - 新增按邮箱查询套餐记录测试。
  - 新增关键字与状态筛选组合测试。

## 核心代码说明

列表接口保持原有分页和状态筛选逻辑，仅在查询构造阶段增加用户条件：

- `keyword=157`：匹配 `SysUser.id == 157`，同时兼容用户名/邮箱中包含 `157` 的记录。
- `keyword=liujixin`：匹配 `SysUser.username LIKE '%liujixin%'` 或 `SysUser.email LIKE '%liujixin%'`。

前端搜索框与状态筛选共用同一个 `fetchList`，查询和重置都会把分页重置到第一页。
