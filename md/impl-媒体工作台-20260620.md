# 媒体工作台实施记录

## 任务概述

用户端新增「媒体工作台」独立页面，用于图片生成、参考图生图、文生视频、图生视频，支持结果预览、下载、原始响应查看，并展示最近 24 小时媒体调用成功率。后端补充 1K 图片请求优先使用 `OpenAI Image Compatible` 渠道的稳定排序规则，2K/4K 继续通过已有能力过滤到 `OpenAI Image Native Size` 渠道。

- `backend/app/api/user/media_workbench.py`：新增用户端媒体工作台健康摘要接口。
- `backend/app/services/media_workbench_service.py`：新增最近调用成功率聚合服务。
- `backend/app/main.py`：注册媒体工作台用户端路由。
- `backend/app/services/proxy_service.py`：新增 1K 图片 compatible 优先排序，并接入图片生成、图片编辑。
- `backend/test_media_workbench.py`：新增图片渠道排序和健康摘要单测。

## 核心代码说明

### 图片渠道路由

- 新增 `ProxyService._prefer_openai_compatible_for_1k_image(channels, image_size)`。
- 仅在 `image_size == "1K"` 时生效。
- 只调整 `openai-image-compatible` 与 `openai-image-native-size` 两类候选在原列表目标槽位内的相对顺序。
- 非 OpenAI 渠道、默认渠道、同类渠道内部顺序保持不变。

### 健康摘要

- `GET /api/user/media-workbench/health?window_hours=24`
- 固定返回：
  - `image_gpt_image_2`
  - `video_grok`
- 生图统计模型：`gpt-image-2`、`codex-gpt-image-2`、`plus-codex-gpt-image-2`。
- 视频统计模型：`grok-imagine-video`。
- 成功条件复用现有 `LogService._visible_success_condition()`。
- 排除明显本地前置错误，例如积分不足、参数错误、模型不存在等。
- 同步视频生成等待失败、等待超时和内容校验失败按 `video_generation` 失败样本记录，确保 Grok 视频健康度不会只统计任务创建结果。
- 不向用户端返回渠道 ID、渠道名称或错误明细。

### 前端工作台

- 页面位于 `/user/media-workbench`。
- 支持生图分辨率选择 `1K/2K/4K` 与比例选择。
- 支持参考图生图，模型支持图片编辑时调用 `/v1/image/edit` 上传参考图；页面最多可选择 7 张参考图，并按重复 `image` 字段提交到后端。
- 支持文生视频、图生视频；图生视频上传参考图。
- 图生视频最多可选择 7 张参考图，并按重复 `input_reference[]` 字段提交到后端。
- 视频生成已由同步 `/v1/created/video` 调整为异步 `/v1/videos` 创建任务、前端轮询 `/v1/videos/{video_id}`，最长等待 300 秒，完成后拉取 `/v1/videos/{video_id}/content` 用于预览和下载。
- 视频预览通过带 Authorization 的 `fetch` 拉取 blob 后转 object URL。
- 页面销毁、清空结果和替换结果时释放 object URL。
- API Key 准备逻辑复用公共 `prepareUserApiKey()`，避免重复创建密钥。

### 计费保障

- 媒体工作台图片生成调用 `/v1/images/generations`，继续进入 `ProxyService.handle_image_request()` 的统一图片代理链路。
- 图片链路只有在上游响应成功、`_parse_openai_image_response()` 解析到有效图片后，才调用 `_deduct_image_credits_and_log()` 扣媒体积分并写成功日志。
- 图片失败、渠道异常、参数错误或结果为空时走 `_log_failed_request(... image_credits_charged=0 ...)`，不扣费。
- 媒体工作台参考图生图调用 `/v1/image/edit`，继续进入 `ProxyService.handle_image_edit_request()`，同样只有解析到有效图片后才扣费。
- 媒体工作台视频生成调用 `/v1/videos` 创建任务，完成后通过 `/v1/videos/{video_id}/content` 拉取内容。
- 视频链路创建任务时 `bill_on_create=False`，内容接口校验视频内容后才调用 `_bill_video_route_if_needed()` / `_log_video_success()`。
- 视频失败、等待失败、内容校验失败或本地异常时写失败日志且 `image_credits_charged=0`，不扣费。
- `/v1/created/video` 的等待失败和内容校验失败统一写入 `request_type=video_generation`；异步视频创建成功但未完成时不扣费，内容获取成功后写成功日志，使媒体工作台健康度统计到用户感知结果。
