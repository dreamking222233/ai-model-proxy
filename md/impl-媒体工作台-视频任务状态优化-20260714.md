## 核心代码说明

- `startVideoTaskTracking()` 启动任务提交阶段和秒级等待计时。
- `updateVideoTaskState()` 将上游状态映射为排队、生成和完成阶段，并只展示上游真实返回的进度。
- `waitForVideoCompletion()` 每次轮询后更新状态卡及原始响应。
- 轮询使用绝对 20 分钟截止时间，最后一次查询的超时不会越过总截止时间。
- `stopVideoTaskTracking()` 统一释放计时器。
- `cancelActiveVideoTask()` 通过运行 ID、可取消 sleep 和 `AbortController` 终止离开页面后的异步链。
- 前端成功/失败终态集合与后端对齐，包含 `done/error/expired`。
- 进度条成功态只由完成状态决定；`in_progress + 100%` 仍显示生成中。
- SQL 使用渠道名称、兼容根地址与 `/v1` 的规范化 Base URL 和模型名三重条件，重复执行结果一致。
