# 安全-模型监控开关 Impl

## 任务概述

实现按统一模型控制安全风控开关：

- `admin/models` 管理单个模型时可开启/关闭“安全监控”。
- 开启安全监控的模型继续执行现有违规检测、拦截、请求快照、风险事件记录、模型安全提示词和输出标记处理。
- 关闭安全监控的模型跳过上述安全风控处理，Grok 系列默认关闭。
- 新建模型默认规则：`gpt`、`claude` 开启；`grok`、`gemini`、`other` 关闭。

## 核心代码说明

### 模型字段

`security_monitor_enabled` 是模型级开关，不替代全局 `security_*` 配置：

- 模型开关关闭：不创建快照、不扫描输入、不注入安全提示词、不处理输出风险标记。
- 模型开关开启：继续受现有全局配置控制，例如 `security_detection_enabled`、`security_snapshot_enabled`、`security_model_prompt_enabled`。

### 代理入口

代理入口从原来的“先风控再解析模型”调整为“先解析目标模型，再根据模型字段决定是否风控”。

这样可以支持同一个代理入口下：

- `gpt` / `claude` 模型拦截风险请求。
- `grok` 模型完全跳过安全风控。

### 输出处理

流式输出缓冲器按模型开关选择：

- 开启：使用 `_SecurityRiskMarkerStreamBuffer` 清洗 `[MIS_RISK_REPORT ...]`。
- 关闭：使用 `_PassthroughTextBuffer` 原样透传。

## 测试验证

## Review 后修复记录

自审发现并修复以下问题：

- Responses WebSocket 转发已接入模型级流式输出缓冲，开启安全监控时会清洗 `[MIS_RISK_REPORT ...]` 并在 `response.completed` 后执行输出扫描；关闭监控时继续原样透传。
- Responses 普通 SSE 的 `response.completed.response` 已复用统一清洗逻辑，避免只清洗 delta 而 completed 对象仍透传内部风险标记；多段 `output[].content[].text` 会逐字段清洗，不合并文本结构。
- Responses、OpenAI、Anthropic 非流式输出清洗改为逐 text 节点移除内部风险标记，风险事件扫描仍基于全文执行，但响应结构不会被合并写回单个节点。
- Responses 生图分流改为按最终解析模型判断，支持请求模型通过 override 指向图片模型时进入图片入口，并由最终图片模型的安全监控开关接管。
- Responses HTTP / WebSocket 在延迟 flush 被保留的 delta 文本时，会复用上一条 `response.output_text.delta` 的上下文字段，避免客户端丢失 output/content 索引。
- 关闭安全监控的 `_PassthroughTextBuffer` 只透传 chunk，不再累计 `raw_text` / `visible_text`，避免关闭监控模型保留完整输出文本。
- 视频 Chat Completions 的 `seconds`、`size`、余额预检和扣费参数改为在渠道循环内按当前渠道计算，避免未定义 `channel`。
- `backend/sql/init.sql`、`backend/sql/initData.sql` 和 `sql/initData.sql` 移除无条件回写安全监控字段的逻辑，`ON DUPLICATE KEY UPDATE` 不覆盖管理员手动设置。
- 升级脚本不再依赖 `long_context_billing_enabled` 的固定列位置；会按 `model_series` 和模型名前缀将 `gpt/o1/o3/o4/claude` 开启安全监控，其余关闭，兼容系列列缺失或历史系列值未回填的库。

### 后端冒烟

```bash
PYTHONPATH=backend python - <<'PY'
from types import SimpleNamespace
from app.services.model_service import ModelService
from app.services.proxy_service import ProxyService

for series in ['gpt', 'claude', 'grok', 'gemini', 'other']:
    print(series, ModelService._normalize_security_monitor_enabled(None, series, None))

for enabled in [1, 0]:
    buf = ProxyService._build_security_stream_buffer(SimpleNamespace(security_monitor_enabled=enabled))
    out = buf.feed('a[MIS_RISK_REPORT category=cyber_abuse severity=high]b') + buf.flush()
    print(enabled, type(buf).__name__, out)
PY
```

结果：

- `gpt`、`claude` 默认 `1`。
- `grok`、`gemini`、`other` 默认 `0`。
- 监控模型流式缓冲会移除内部风险标记。
- 放开模型流式缓冲原样透传。
