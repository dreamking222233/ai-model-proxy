# 安全-登录与CORS加固 Impl

## 任务概述

针对登录凭证、CORS 任意源反射、注册/登录暴力请求、Source Map 公开、安全响应头、nginx 版本暴露、邮箱验证、JWT 单独吊销能力进行加固。同时修复前端生产环境 API 地址写死 `https://api.xiaoleai.team` 导致复制部署误连平台 API 的问题。

## 核心代码说明

1. CORS
   - 后端只保留 `DynamicCORSMiddleware`，允许来源由平台配置、正则和代理前台域名决定。
   - nginx 不再覆盖后端 CORS 响应，避免任意 Origin + Credentials。

2. Token
   - 新 token 包含 `sub`、`tv`、`exp`、`iat`、`jti`。
   - 请求鉴权时校验 token 中 `tv` 与 `sys_user.token_version` 一致。
   - 通过递增 `token_version` 实现账号级旧 token 失效。
   - 登出时按 `jti` 写入撤销缓存，支持单 token 失效。

3. 限流
   - 登录按 IP + 用户名，每分钟默认 10 次。
   - 注册、忘记密码、邮箱验证码按 IP + 账号/邮箱，每小时默认 5 次。
   - Redis 可用时使用 Redis 计数；不可用时使用进程内窗口降级。
   - 限流 IP 不再信任客户端伪造的 `X-Forwarded-For`，由 nginx 覆盖转发真实来源。

4. 邮箱验证
   - 邮件服务未接通前默认 `EMAIL_VERIFICATION_REQUIRED=false`，生产接通 SMTP 后可改为 `true`。
   - SMTP 未配置时发送验证码接口返回 503，避免假发送或绕过。
   - 验证码只保存哈希，不保存明文。
   - 找回密码验证码发送前先校验账号与邮箱匹配。
   - 前端通过公开站点配置决定验证码字段是否必填。

5. 前端部署
   - 生产 API 不再固定到 `api.xiaoleai.team`。
   - 如果部署为同源反代，`VUE_APP_BASE_API` 可留空；如果前后端分域，构建时设置目标 API 域名。

## 部署注意事项

- 生产环境需要执行 `backend/sql/upgrade_security_auth_hardening_20260524.sql`。
- 生产环境必须配置强 `JWT_SECRET_KEY`。
- 启用注册/找回密码前需要配置 `SMTP_HOST`、`SMTP_FROM`，如需认证再配置 `SMTP_USERNAME`、`SMTP_PASSWORD`。
- 复制部署到 `zelong.vip` 这类独立域名时：
  - 推荐同源反代 `/api`、`/v1` 到后端，前端 `VUE_APP_BASE_API` 留空。
  - 如果分域部署，构建前设置 `VUE_APP_BASE_API=https://你的API域名`。
- `server_tokens off;` 最好放在 nginx 全局 `http` 块，当前 server 块配置用于模板兜底。
