# 安全-违规请求检测上报 实施记录

## 任务概述

根据 `plan-安全-违规请求检测上报-20260618.md` 实施安全风控闭环：

- 请求进入模型代理层后创建短期请求快照。
- 通过本地关键词和组合语义规则进行常规检测，高风险请求在上游调用前阻断并记录。
- 向文本模型注入安全拒绝和风险上报提示词，包含公开上报地址、`snapshot_id`、短期 `report_token`。
- 支持模型通过公开接口上报；模型无法调用 HTTP 时，可输出内部标记 `[MIS_RISK_REPORT ...]`，后端捕获并清洗。
- 管理后台新增安全风控页面，可查看风险事件、完整请求快照、命中规则、模型输出片段并处理状态。
- 无风险请求完整体只短期保留，到期后清空完整内容；风险请求标记为 `flagged`，避免被短 TTL 清理。

## 核心逻辑说明

### 常规检测

`SecurityDetectionService.scan_text` 维护结构化规则，覆盖：

- 黄色/色情内容。
- 破限、越狱、绕过系统提示词。
- 注册机、破解网站、爆破、撞库、盗号、验证码/风控绕过等网络滥用。
- 批量注册、接码、代理池、刷量、异常自动化。
- “我是学生/学校作业/仅供学习/老师要求”等前置理由叠加注册机、破解、异常自动化时，提升为高风险阻断。

高风险请求由 `ensure_allowed_or_raise` 抛出：

```text
ServiceException(403, security_block_message, "SECURITY_RISK_BLOCKED")
```

### 快照和保留策略

- `security_request_snapshot` 保存短期脱敏请求体、提取文本、请求 hash、用户、模型、协议、IP、过期时间。
- 敏感字段如 token、password、cookie、secret 会被替换为 `***`。
- base64、图片、文件、音视频字段只保存摘要、大小和 hash。
- 超过 `security_snapshot_max_body_bytes` 会截断并设置 `is_truncated=1`。
- 未命中风险的快照按 `security_snapshot_ttl_seconds` 到期后清空 `request_body_json` 和 `extracted_text`。
- 命中风险后快照标记为 `flagged`，用于管理员查看完整内容。

### 模型拒绝和上报

文本请求注入安全系统提示词，要求模型：

- 对黄色、破限、注册机、破解网站、异常自动化、非法攻击等请求拒绝。
- 对学生、作业、仅供学习等理由不放行。
- 有 HTTP 能力时提交：

```text
POST https://api.xiaoleai.team/api/public/security/risk-report
```

- 无 HTTP 能力时输出内部标记：

```text
[MIS_RISK_REPORT category=... severity=...]
```

后端在非流式和流式响应中捕获该标记，记录 `security_risk_event`，并从用户可见响应中移除标记。

### 公开上报接口

接口：`POST /api/public/security/risk-report`

认证策略：

- 不依赖登录态，满足“放开权限”。
- 必须校验后端生成的 `snapshot_id + report_token`。
- 过期、缺失、伪造 token 均拒绝。
- 同一快照、分类、来源去重。

### 管理端

新增后台页面 `/admin/security-risks`，支持：

- 查看待处理、高风险、今日风险、已清理快照统计。
- 按用户 ID、风险分类、风险等级、来源、状态筛选。
- 查看风险事件详情、命中规则、提取文本、完整请求体、模型输出片段。
- 更新处理状态和备注。
- 手动清理过期无风险快照。

## Review 后修复记录

根据 `md/review-安全-违规请求检测上报-20260618.md` 完成以下修复：

- 高风险输入命中时，即使风险事件写入失败，也继续抛出 `SECURITY_RISK_BLOCKED`，避免风控写库异常导致高风险请求放行。
- Responses 图片分支复用外层 `request_id`，保证安全快照、响应头和实际图片请求日志可关联。
- Responses WebSocket 只在安全扫描通过后更新 `last_request`，避免被拦截轮次污染后续会话状态。
- 流式内部标记过滤改为状态机，遇到 `[MIS_RISK_REPORT` 后持续吞掉直到 `]` 或安全长度上限，避免长 `reason` 跨 chunk 泄漏。
- `security_flagged_retention_days` 已接入清理逻辑，超过保留天数的 flagged 快照会清空完整请求体并保留事件元数据。
- ORM 对齐 MySQL SQL：`request_body_json` 使用 `LONGTEXT` 变体，`extracted_text` 和 `response_excerpt` 使用 `MEDIUMTEXT` 变体。
