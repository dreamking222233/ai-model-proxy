# 安全风控-序列化会话误判 Impl

## 任务概述

修复用户 `DDD` 的安全风控误判。生产样本显示 `keygen` 来自正常 `ssh-keygen` 命令，“后门”来自“没发现后门”的源码审计结论。实现改为关键词上下文消歧，不信任客户端序列化会话的角色字段。

## 文件变更

- `backend/app/services/security_detection_service.py`
  - `keygen` 从无条件词表转为上下文正则，排除 `ssh-keygen`。
  - 制作、生成、破解、激活、授权绕过等 `keygen` 滥用语境仍是 `high/block`。
  - “后门”规则按每一次词语出现位置识别“没发现/未发现/不存在”等否定式审计语境，否定范围不能跨句。
  - 保留整段外层用户文本扫描，伪造 JSON `role` 不能绕过。
- `backend/app/services/proxy_service.py`
  - 恢复被后续视频提交覆盖的请求扫描固定 fail-closed 逻辑。
- `backend/test_security_detection_service.py`
  - 新增 `ssh-keygen`、否定式后门审计、真实 `keygen` 滥用、否定与植入后门混合文本和 JSON 角色伪造回归测试。

## 安全边界

客户端把整个会话 JSON 放入单个 `user` 消息时，内层 `role` 不是可信边界。因此实现不根据 inner role 忽略内容，而是对完整文本执行上下文规则，同时兼顾误判和绕过风险。
