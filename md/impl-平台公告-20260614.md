# 平台公告功能实施记录

## 任务概述

新增管理端公告管理页面，支持编辑直营客户固定开屏公告、发布额外平台公告；用户端登录后按顺序弹出当前站点固定公告与额外公告；用户端 Header 头像左侧增加公告图标，可查看当前平台公告内容。

## 核心代码说明

1. 固定开屏公告继续使用 `system_config` 的 `platform_announcement_title`、`platform_announcement_content`、`platform_support_wechat`、`platform_support_qq`，只影响直营客户。
2. 额外公告存储在 `platform_announcement` 表，发布后通过 `GET /api/user/profile/announcements` 面向全部用户返回，包括代理端用户。
3. 用户公告接口返回当前站点固定公告：
   - 直营站点：管理端固定开屏公告。
   - 代理站点：代理端自定义固定开屏公告。
   再追加管理端已发布额外公告。
4. Dashboard 使用 `sessionStorage` 按公告 ID 记录当前会话已弹出状态，支持固定公告确认后继续弹出下一条额外公告。
5. Header 公告抽屉展示接口返回的所有当前公告，便于用户后续主动查看。
6. 开屏公告禁用遮罩关闭，避免用户误关闭后跳过后续公告队列。
