# 推广关系手动绑定实施记录

## 任务概述

本次调整推广返现比例，并为管理端补充手动绑定推广关系能力：

- 推广返现比例从 50% 调整为 20%。
- 管理端新增“手动绑定”入口，管理员可指定推广人和被推广人。
- 手动绑定只影响后续充值返现，不回溯历史订单。

## 核心代码说明

### 1. 返现比例

`PromotionService.REWARD_RATE` 从 `Decimal("0.5")` 调整为 `Decimal("0.2")`。

该比例只用于新充值订单生成返现时计算：

- 余额返现：`credited_usd * 0.2`
- 图片积分返现：`credited_image_credits * 0.2`

历史 `user_promotion_reward` 记录保持不变。

返现服务层同时增加保护：

- 非 `paid` 状态或缺少 `paid_at` 的订单不返现。
- 订单创建时间早于推广关系创建时间时不返现，避免手动补绑后回溯历史订单。

### 2. 管理端手动绑定

新增服务方法：

- `PromotionService.manual_bind_relation(db, promoter_user_id, invited_user_id)`

校验规则：

- 推广人和被推广人都必须是正常普通用户。
- 不允许自己绑定自己。
- 被推广人已有绑定关系时拒绝。
- 直营/代理站点必须一致，禁止跨站点、跨代理绑定。
- 推广人没有推广链接时自动懒创建。
- 手动绑定前已经创建的订单不会因为补绑而产生返现。

新增接口：

- `POST /api/admin/promotions/manual-bind`

请求体：

```json
{
  "promoter_user_id": 1,
  "invited_user_id": 2
}
```

### 3. 管理端页面

`admin/promotions` 的“推广关系”工具栏新增“手动绑定”按钮。

弹窗字段：

- 推广人用户 ID：获得返现的用户。
- 被推广人用户 ID：后续充值触发返现的用户。

绑定成功后自动刷新汇总、推广关系列表和返现明细。
