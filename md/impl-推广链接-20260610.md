# 推广链接实施记录

## 任务概述

本次实现新增了用户推广链接体系，覆盖以下能力：

- 普通用户生成专属推广链接，直营站与代理站按各自域名生成注册链接。
- 新用户通过 `invite_code` 注册时绑定推广关系，非法/跨站推广码会直接阻断注册并整体回滚。
- 被推广用户在线充值成功后，推广人按到账资产获得 50% 返现：
  - 余额充值返现到美元余额
  - 图片积分充值返现到图片积分余额
- 用户端新增推广页面，可查看推广链接、注册用户、是否充值、累计返现。
- 管理端与代理端新增推广记录页面，可查看推广关系、返现明细与顶部聚合统计。
- `user/asset-source` 扩展为支持余额与图片积分统一来源查询，并展示推广返现。

## 核心代码说明

### 1. 推广主数据模型

新增三张表：

- `user_promotion_link`：用户固定推广码与统计冗余字段
- `user_promotion_relation`：推广人和被推广人的绑定关系
- `user_promotion_reward`：每笔充值生成的返现明细

`user_promotion_reward` 使用 `(order_no, reward_asset_type)` 唯一约束，确保充值回调重复触发时不会重复返现。

### 2. 注册绑定事务

`AuthService.register()` 已新增 `invite_code` 入参，用户、初始余额、图片积分账户创建后先 `flush()`，再调用 `PromotionService.bind_invited_user()`。

绑定规则：

- 仅普通用户可参与推广关系
- 非法/停用推广码报错
- 直营推广码不能绑定代理用户
- 代理推广码只能绑定同代理用户
- 不允许自己推广自己

绑定失败会直接抛 `ServiceException`，整个注册事务回滚。

### 3. 支付返现接入点

在 `PaymentService._apply_paid_order()` 的首次成功入账分支内，现有流程顺序为：

1. 用户充值到账
2. 代理现金分润
3. 推广返现
4. 更新充值 settlement

推广返现逻辑集中在 `PromotionService.apply_recharge_reward()`：

- 余额充值：返现 `credited_usd * 0.5`
- 图片积分充值：返现 `credited_image_credits * 0.5`
- 返现只增加可用余额，不增加 `total_recharged`
- 同时写入余额或图片积分明细流水
- 更新推广关系与推广链接冗余统计字段

### 4. 资产来源统一查询

`BalanceService.get_asset_source_records()` 已扩展支持：

- `asset_type=all|balance|image_credit`
- 余额流水与图片积分流水统一分页
- 推广返现优先按 `user_promotion_reward` 结构化记录识别

前端 `AssetSource.vue` 同步增加：

- 余额 / 图片积分汇总卡片
- 资产类型筛选
- 图片积分单位展示

### 5. 三端推广页面

- 用户端：推广链接、复制、注册与返现明细
- 管理端：推广关系表 + 返现明细表，支持筛选，并展示顶部汇总统计
- 代理端：仅查看当前代理范围内的推广关系和返现明细，并展示顶部汇总统计

### 6. 注册页邀请码透传

`Login.vue` 在注册提交时会读取当前路由的 `invite_code` 并传给后端，同时在登录/注册模式切换时保留 query，避免链接参数丢失。
