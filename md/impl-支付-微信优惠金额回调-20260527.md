# 支付-微信优惠金额回调 Impl

## 任务概述

修复微信支付用户使用优惠后，回调/查单因 `payer_total` 小于本地订单金额而误判金额不一致的问题。微信金额校验改为优先使用商户订单金额 `amount.total`，保证原价支付和优惠支付都能正常入账。

同时在用户充值页右侧增加“支付后未到账”提醒，展示当前站点微信和 QQ 联系方式。代理站点继续复用现有站点配置接口返回的代理联系方式。

## 文件变更清单

- `backend/app/services/payment_service.py`
  - 微信支付成功 payload 校验优先使用 `amount.total`。
  - 保留 `amount.payer_total` 兜底兼容旧测试或异常 payload。
- `backend/app/test/test_payment_recharge.py`
  - 新增微信优惠支付测试：本地订单 10 元、`total=1000`、`payer_total=604` 时应成功入账。
  - 新增微信金额兜底测试：`amount.total` 缺失但 `payer_total` 与本地订单一致时应成功入账。
  - 新增反向测试：`total` 与本地订单不一致时，即使 `payer_total` 匹配也拒绝入账。
  - 新增异常格式测试：微信 `amount` 不是对象时拒绝入账并保持订单待支付。
  - 新增缺失字段测试：微信成功 payload 缺失 `amount` 或 `amount={}` 时拒绝入账。
- `frontend/src/views/user/Recharge.vue`
  - 右侧支付说明卡片新增未到账提醒。
  - 新增 `supportWechat`、`supportQq` 计算属性展示当前站点联系方式，并让提醒区和客服区统一复用。
  - 联系方式未配置时明确展示“未配置”。

## 核心代码说明

微信交易金额字段语义：

- `amount.total`：商户订单总金额，系统应与本地 `amount_cny` 对账。
- `amount.payer_total`：用户扣除微信/银行优惠后的实际支付金额，不能用于判断本地订单是否被篡改。

因此本次入账校验从旧逻辑：

```python
amount_data.get("payer_total") or amount_data.get("total")
```

调整为：

```python
amount_data.get("total")
```

仅当 `total` 缺失时再兜底读取 `payer_total`。

同时对微信金额字段增加格式异常包装，避免异常 payload 绕过统一业务异常处理。若 `amount` 缺失、不是对象、为空对象、缺少 `total/payer_total`，或金额值无法转换为分金额，均以 `WECHAT_PAY_AMOUNT_MISMATCH` 拒绝入账。
