# 微信支付配置说明

本文档说明当前微信 Native 支付代码落地后，本地或线上部署时需要补充的配置项。

## 需要执行的 SQL

新增微信支付字段需要先执行以下脚本：

`backend/sql/upgrade_wechat_native_recharge_20260520.sql`

如果是全量初始化数据库，还需要确保 `sql/initData.sql` 与最新表结构保持一致。

## 必填配置项

以下配置写入 `backend/.env`，不要提交到 Git：

```env
WECHAT_PAY_ENABLED=true
WECHAT_PAY_SERVER_URL=https://api.mch.weixin.qq.com
WECHAT_PAY_APP_ID=
WECHAT_PAY_MCH_ID=
WECHAT_PAY_SERIAL_NO=
WECHAT_PAY_PRIVATE_KEY=
WECHAT_PAY_API_V3_KEY=
WECHAT_PAY_NOTIFY_URL=
WECHAT_PAY_PUBLIC_KEY_ID=
WECHAT_PAY_PLATFORM_PUBLIC_KEY=
```

各字段说明：

- `WECHAT_PAY_ENABLED`：是否启用微信支付。
- `WECHAT_PAY_SERVER_URL`：微信支付 API v3 服务地址，生产一般固定为 `https://api.mch.weixin.qq.com`。
- `WECHAT_PAY_APP_ID`：微信支付绑定的应用 AppId。
- `WECHAT_PAY_MCH_ID`：微信支付商户号。
- `WECHAT_PAY_SERIAL_NO`：商户 API 证书序列号，用于请求签名。
- `WECHAT_PAY_PRIVATE_KEY`：商户 API 证书私钥内容，建议保存为单行 `\n` 转义格式。
- `WECHAT_PAY_API_V3_KEY`：APIv3 密钥，用于解密微信回调报文。
- `WECHAT_PAY_NOTIFY_URL`：微信支付异步回调地址，必须能被微信公网访问。
- `WECHAT_PAY_PUBLIC_KEY_ID`：微信支付公钥 ID，灰度切换公钥模式时用于请求头 `Wechatpay-Serial`。
- `WECHAT_PAY_PLATFORM_PUBLIC_KEY`：微信支付公钥内容，建议保存为单行 `\n` 转义格式。

## 灰度切换期间额外配置

如果商户号正处于“平台证书切换到微信支付公钥”的灰度期，回调可能仍然使用平台证书签名。此时还需要配置：

```env
WECHAT_PAY_PLATFORM_CERT=
WECHAT_PAY_PLATFORM_SERIAL=
```

各字段说明：

- `WECHAT_PAY_PLATFORM_CERT`：微信支付平台证书 PEM 内容。当前项目支持直接填写证书 PEM。
- `WECHAT_PAY_PLATFORM_SERIAL`：平台证书序列号。当前代码在请求头优先发送 `WECHAT_PAY_PUBLIC_KEY_ID`，未配置时才回退到这个字段。

## 当前代码行为

- 下单请求会自动带 `Wechatpay-Serial`，优先使用 `WECHAT_PAY_PUBLIC_KEY_ID`。
- 回调验签会根据回调头 `Wechatpay-Serial` 自动判断：
  - `PUB_KEY_ID_...`：使用 `WECHAT_PAY_PLATFORM_PUBLIC_KEY` 验签。
  - 其他序列号：使用 `WECHAT_PAY_PLATFORM_CERT` 验签。
- 前端已改为“立即支付”，微信支付成功后会自动轮询后端 `/sync` 接口，无需手动点检查状态。

## 风险说明

如果只配置微信支付公钥，没有配置平台证书，而商户号又处在微信官方灰度期，则会出现：

- 下单成功
- 前端自动轮询可主动查单并完成入账
- 但异步回调会因平台证书验签缺失而持续报错

因此正式上线前，建议同时配置 `WECHAT_PAY_PLATFORM_CERT`，直到商户平台显示回调公钥切换比例达到 100%。
