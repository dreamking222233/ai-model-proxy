# 支付-支付宝与代理在线充值交付说明

日期：20260516

## 1. 本次交付范围

本次已完成两组能力：

### 1.1 支付宝在线充值

覆盖三端能力：

1. 用户端新增在线充值页面
2. 后端接入支付宝 PC 网页支付
3. 支付成功后，用户余额自动增加到现有系统余额体系，可继续调用 API
4. 代理站用户充值后，代理现金余额按分润规则累计
5. 管理端新增代理现金余额、提现管理页面
6. 代理端新增用户充值记录与代理现金余额展示页面

### 1.2 管理端控制代理在线充值开关

1. 管理端可按代理开启/关闭在线充值
2. 代理站关闭后：
   - 用户端不显示“在线充值”菜单
   - 账单页不显示“立即充值”按钮
   - 手动访问 `/user/recharge` 会显示禁用提示
   - 后端接口禁止继续创建充值订单

## 2. 业务规则

### 2.1 用户充值比例

- `1 RMB = 5 USD`

### 2.2 代理结算比例

- 代理内部结算比例：`1 RMB = 10 USD`

### 2.3 代理现金分润公式

当代理用户充值 `amount_cny` 时：

1. 用户到账美元：
   - `credited_usd = amount_cny * 5`
2. 代理内部成本人民币：
   - `agent_cost_cny = credited_usd / 10`
3. 代理现金分润人民币：
   - `agent_income_cny = amount_cny - agent_cost_cny`

示例：

- 用户在代理站充值 `10 RMB`
- 用户到账 `50 USD`
- 代理成本 `5 RMB`
- 代理现金余额 `+5 RMB`

## 5. 服务器需要补充的支付宝配置

注意：

1. 这些配置不会提交到 GitHub
2. 你需要在服务器本地的 `backend/.env` 中手动填写
3. 当前后端配置文件会自动读取 `backend/.env`

### 5.1 必填环境变量

```env
ALIPAY_ENABLED=true
ALIPAY_SERVER_URL=https://openapi.alipay.com/gateway.do
ALIPAY_APP_ID=你的支付宝应用APPID
ALIPAY_APP_PRIVATE_KEY=你的应用私钥
ALIPAY_PUBLIC_KEY=支付宝公钥
ALIPAY_SIGN_TYPE=RSA2
ALIPAY_NOTIFY_URL=https://你的公网API域名/api/public/payment/alipay/notify
ALIPAY_RETURN_PATH=/user/recharge
PAYMENT_PUBLIC_BASE_URL=https://你的公网API域名
RECHARGE_USER_CNY_TO_USD_RATE=5
RECHARGE_AGENT_CNY_TO_USD_SETTLEMENT_RATE=10
```

### 5.2 字段说明

#### `ALIPAY_ENABLED`

- 是否开启在线充值
- `true` 表示后端允许创建支付宝充值订单
- 若为 `false`，平台和代理站都不能发起在线充值

#### `ALIPAY_SERVER_URL`

- 支付宝网关地址
- 正式环境使用：
  - `https://openapi.alipay.com/gateway.do`

#### `ALIPAY_APP_ID`

- 支付宝开放平台应用 ID
- 来源：支付宝开放平台应用详情页

#### `ALIPAY_APP_PRIVATE_KEY`

- 应用私钥
- 来源：你自己的应用密钥
- 建议直接保存完整 PEM 内容，或保存单行原始 Key 字符串
- 当前代码会自动兼容常见格式，并转换为 SDK 可用格式

#### `ALIPAY_PUBLIC_KEY`

- 支付宝公钥
- 来源：支付宝开放平台应用密钥设置页

#### `ALIPAY_SIGN_TYPE`

- 签名方式
- 当前固定建议：
  - `RSA2`

#### `ALIPAY_NOTIFY_URL`

- 支付宝异步通知地址
- 必须是支付宝服务器可以访问的公网 HTTPS 地址
- 示例格式：
  - `https://api.example.com/api/public/payment/alipay/notify`

#### `ALIPAY_RETURN_PATH`

- 支付完成后的前端回跳路径
- 当前固定页面为：
  - `/user/recharge`

#### `PAYMENT_PUBLIC_BASE_URL`

- 后端公网访问基地址
- 当 `ALIPAY_NOTIFY_URL` 未单独填写时，会基于这个地址拼出默认通知地址
- 示例：
  - `https://api.example.com`

#### `RECHARGE_USER_CNY_TO_USD_RATE`

- 用户充值汇率
- 当前按需求固定：
  - `5`

#### `RECHARGE_AGENT_CNY_TO_USD_SETTLEMENT_RATE`

- 代理内部结算汇率
- 当前按需求固定：
  - `10`

## 6. 与域名相关的附加配置

为了让支付宝回跳、代理站识别、以及多站点前端返回逻辑正常工作，服务器还需要确认这些配置与实际域名一致。

### 6.1 平台前端域名

配置项：

- `PLATFORM_FRONTEND_HOSTS`

要求：

1. 需要包含平台前端域名
2. 若本地也要调试，可保留 `localhost`

### 6.2 平台 API 域名

配置项：

- `PLATFORM_API_HOSTS`

要求：

1. 需要包含平台后端公网 API 域名
2. 支付宝通知打到这个域名时，后端才能正确识别平台上下文

### 6.3 代理域名

代理相关字段在管理端维护：

- `frontend_domain`
- `api_domain`

要求：

1. 代理前台域名必须能被前端真实访问
2. 若代理通过共享 API 访问后端，前端请求会自动带 `X-Site-Host`
3. 后端会据此识别当前站点归属，并生成正确回跳地址

## 7. 支付宝开放平台侧需要确认的配置

在支付宝应用后台，需要确认以下项目：

1. 应用已上线或具备可支付能力
2. 签名方式为 `RSA2`
3. 应用私钥与支付宝公钥对应当前应用
4. 异步通知地址与服务器 `ALIPAY_NOTIFY_URL` 一致
5. 若后台要求配置回调白名单或 HTTPS 证书，需与线上域名一致
