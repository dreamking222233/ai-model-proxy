## 核心代码说明

### 1. 初始化数据补充

在最终初始化脚本末尾、`SET FOREIGN_KEY_CHECKS = 1;` 之前补充了以下初始化数据：

- 默认管理员账户
- 管理员余额记录
- 管理员图片积分余额记录
- `system_config` 默认配置

这样可以保证执行该脚本时，除了表结构外，还能落下最基本的启动配置数据。

### 1.1 初始化入口收敛

完成结构同步和初始化数据补充后，将完整脚本收敛为根目录唯一初始化脚本 `sql/initData.sql`，并删除重复的 `sql/ai_model_proxy.sql`，避免后续双文件分叉。

### 2. 导出噪音清理

为了让文件适合长期维护和提交，移除了以下不必要信息：

- Navicat 导出头注释
- 所有表定义中的 `AUTO_INCREMENT=...` 当前值
- 两处 `utf8mb4_0900_ai_ci`，统一为 `utf8mb4_unicode_ci`

保留内容：

- `SET NAMES utf8mb4;`
- `SET FOREIGN_KEY_CHECKS = 0;`
- 基于当前数据库 `DATABASE()` 的旧视图清理逻辑
- `DROP TABLE IF EXISTS ...`
- 全部正式环境表结构定义
- `SET FOREIGN_KEY_CHECKS = 1;`
