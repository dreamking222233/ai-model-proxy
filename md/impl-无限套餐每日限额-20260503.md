# 无限套餐每日限额 Impl

## 任务概述

为无限日包、周包、月包增加每日 2 亿 raw token 使用上限。限制在实际请求链路生效，覆盖管理员模板发放、管理员旧版无限开通、代理端套餐发放等入口。无限套餐超额后直接返回“已超出实际使用额度”，不走余额兜底。

## 文件变更清单

- `backend/app/services/subscription_service.py`
  - 新增 `UNLIMITED_DAILY_TOKEN_LIMIT = 200000000` 和专用错误码。
  - 将 `unlimited` 纳入每日周期校验，复用 `subscription_usage_cycle`。
  - unlimited 周期口径固定为 `total_tokens`，额度快照固定为 2 亿 token。
  - 套餐摘要和记录列表对 unlimited 返回 `current_cycle` 和 `unlimited_daily_token_limit`。
- `backend/app/services/proxy_service.py`
  - 请求前对 active unlimited 套餐执行 quota precheck。
  - 请求后对 unlimited 套餐写入周期用量。
  - unlimited 超额不允许 fallback 到余额计费，并保留明确业务错误。
- `backend/app/test/test_subscription_compatibility.py`
  - 更新 unlimited 套餐预检行为测试。
  - 新增 unlimited 超额不走余额兜底测试。
  - 新增 unlimited 每日周期消费记录测试。
- `frontend/src/views/admin/SubscriptionManage.vue`
  - 套餐模板、套餐记录和旧版无限开通入口展示每日 2 亿 Token 上限。
- `frontend/src/views/agent/SubscriptionManage.vue`
  - 套餐库存和发放记录中展示无限套餐每日 2 亿 Token 上限，并把无限套餐类型文案从“时间套餐”调整为“无限套餐”。
- `md/plan-无限套餐每日限额-20260503.md`

## 核心代码说明

- `SubscriptionService._requires_daily_cycle()` 现在将 `unlimited` 和 `daily_quota` 都视作需要每日周期账本的套餐。
- `SubscriptionService._get_effective_quota_limit()` 对 unlimited 固定返回 `Decimal("200000000")`，不依赖历史套餐记录中的 `quota_value=0`。
- `SubscriptionService.check_quota_before_request()` 会在请求前根据估算 token 判断是否超过剩余额度。
- `SubscriptionService.consume_quota_after_request()` 会在请求成功后按实际 raw token 累加到当天周期。
- `ProxyService._assert_text_request_allowed()` 对 unlimited 超额直接抛出专用错误，不检查余额。
- `ProxyService._deduct_balance_and_log()` 对 unlimited 的请求后超额也直接抛出，不切换到 `balance` billing。

## 待优化项

- 流式请求如果没有显式 `max_tokens`，请求前只能按输入 token 估算；如果输出后才越过 2 亿，系统会在请求后记账阶段拒绝并记录失败，但流式响应可能已经发给客户端。这是现有流式计费架构限制，后续可考虑对 unlimited 用户强制注入输出上限或做更细粒度流式中断。
