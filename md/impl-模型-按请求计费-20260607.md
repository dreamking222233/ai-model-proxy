# 模型-按请求计费 Impl

## 任务概述

为文本模型新增按请求次数计费能力。管理员可在 `admin/models` 新增或编辑模型时选择 `按请求次数计费`，并配置每次请求固定美元价格。请求成功后系统按单次价格计费，同时复用现有系统价格倍率、Responses Fast 倍率和长上下文倍率。超过 25w 左右长上下文阈值时，按次价格同样会被长上下文倍率放大。

## 核心代码说明

按次计费使用新的模型字段：

- `billing_type = request`
- `request_price = 单次请求美元价格`

扣费公式：

```text
total_cost = request_price * price_multiplier * fast_price_multiplier * context_price_multiplier
```

长上下文倍率沿用现有判断：

```text
context_tokens_snapshot > _LONG_CONTEXT_TOKEN_THRESHOLD
```

因此当请求上下文超过长上下文阈值时，按次计费也会自动叠加原有 x2 逻辑。

余额/套餐处理：

- 余额用户：按次计费预检会要求余额足额覆盖 `request_price * 综合价格倍率`；最终本地扣费也会再次校验余额不少于扣费金额。
- 当请求没有明确输出上限时，按次计费预检会保守使用长上下文倍率，避免上游成功后因实际输出触发 x2 而本地记账失败。
- 套餐用户：Token 额度套餐仍按实际 token 消耗额度；如果进入现有 mixed 计费分支，余额补扣沿用当前系统“按未覆盖额度比例折算费用”的策略。
- 对按次计费的 mixed 分支，未覆盖费用必须由余额足额承担；套餐剩余额度不足且余额不足时会在请求前或本地记账阶段拦截，不再允许套餐超额覆盖。
- 成本额度套餐：按 `request_price * fast_price_multiplier * context_price_multiplier` 消耗套餐成本额度，保持不叠加平台 `price_multiplier` 的既有规则。
