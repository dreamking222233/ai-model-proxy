# 模型上下文请求前拦截实施记录

## 任务概述

用户请求到达系统后，先按系统中模型配置的上下文限制进行本地预检。若请求上下文估算超过模型限制，则直接返回 `请求超出最大上下文,请压缩对话`，不再转发给上游，避免上游计费损失。

## 核心实现

- 在 `ProxyService._validate_request_length()` 中恢复模型上下文窗口校验。
- 有效限制取模型 `max_tokens` 与系统配置 `max_context_tokens` 的有效较小值；若只配置其中一个，则使用已配置值。
- 新增上下文专用估算：只统计真实上下文文本，例如 `messages`、`input`、`instructions`、`system`。
- 不统计 `tools`、`metadata`、`reasoning`、`include` 等协议元数据或工具 schema，避免 Codex/Claude Code 请求被误杀。
- OpenAI、Anthropic、Responses 入口均在上游请求前完成校验；OpenAI/Anthropic 已调整为先校验上下文，再做额度预检。
