# 模型映射默认推理强度 Impl

## 任务概述

在模型渠道映射中增加默认推理强度配置。管理员在 `admin/models` 新增映射时可选择 `minimal/low/medium/high/xhigh`；OpenAI Responses 请求保留用户显式 `reasoning.effort`，未设置时使用映射默认值；Anthropic `/v1/messages` 映射到 GPT 上游时优先用户显式值，其次使用映射默认值。

## 文件变更清单

- `backend/app/models/model.py`：`model_channel_mapping` ORM 增加 `default_reasoning_effort`。
- `backend/app/schemas/model.py`：映射创建和返回 schema 增加 `default_reasoning_effort`。
- `backend/app/services/model_service.py`：新增推理强度校验，映射创建、更新、列表返回支持该字段。
- `backend/app/services/proxy_service.py`：Responses 和 Anthropic-to-GPT 路径读取映射默认推理强度并补入上游请求 JSON。
- `backend/sql/init.sql`、`backend/sql/initData.sql`：初始化表结构增加字段。
- `backend/sql/upgrade_model_mapping_default_reasoning_effort_20260601.sql`：新增线上升级脚本。
- `frontend/src/views/admin/ModelManage.vue`：新增映射弹窗选择框和列表展示列。

## 核心代码说明

- `ModelService.REASONING_EFFORT_VALUES` 统一限制管理员可配置值为 `minimal/low/medium/high/xhigh`。
- `ProxyService._apply_responses_mapping_default_reasoning_effort()` 只在 GPT-like 上游且用户没有显式 `reasoning.effort` 时写入映射默认值。
- `ProxyService._apply_anthropic_passthrough_reasoning_effort()` 用于 Anthropic `/v1/messages` 映射 GPT 上游，优先级为：合法用户显式 `reasoning.effort`、Anthropic `thinking` 映射、映射默认值、历史 header 兼容兜底。
- `ProxyService._convert_anthropic_request_to_responses()` 也接收映射默认推理强度，覆盖 `responses:` 显式映射和基于渠道 base_url 的隐式 Responses bridge。
- 用户可见响应仍沿用现有模型名清洗逻辑，不暴露映射后的上游 GPT 模型名。

