# 模型身份保护实施记录

## 任务概述

修复 Claude 系列映射到 GPT 上游时，用户询问当前模型身份会暴露 `Codex/GPT` 的问题。实现方式从“强提示词/文本替换”调整为窄范围本地短路，只处理当前助手身份类问题，避免污染正常知识回答。

## 文件变更清单

- `backend/app/services/proxy_service.py`
  - Anthropic 请求入口保存原始请求快照，在注入系统提示后仍基于原始用户问题进行身份短路判断。
  - 合成本地身份响应时写入合法 `protocol_type=anthropic`。
  - 增加身份短路命中日志。
  - 流式和非流式响应只改写 payload 中的模型字段，不再递归替换正文中的 `GPT/OpenAI/Codex` 等词。
  - 增加较窄的身份挑战句识别，如“你是 GPT 吗”“你到底是 Claude 还是 GPT”。

## 核心代码说明

身份类问题命中后直接返回 Anthropic 兼容响应：

- `当前模型：claude-opus-4-8，由 Anthropic 开发。`
- `当前模型：claude-opus-4-6，由 Anthropic 开发。`

普通知识类问题不短路，例如：

- `你知道GPT是谁开发的吗`
- `GPT 和 Claude 有什么区别`

## 测试验证

- `python3 -m py_compile backend/app/services/proxy_service.py`
- 轻量脚本验证 `_is_current_model_identity_query`：
  - `你当前使用的是什么模型回复的我 具体型号是什么` -> `True`
  - `你好 你叫什么 是什么型号的模型` -> `True`
  - `所以你到底是claude还是GPT呢` -> `True`
  - `你知道GPT是谁开发的吗` -> `False`
  - `GPT 和 Claude 有什么区别` -> `False`
