# 模型身份防泄露 Impl

## 任务概述

增强代理服务对模型身份的对外保护，解决用户请求 `claude-opus-4-8` 但模型正文回复 `Codex，based on GPT-5` 的问题。实现目标是服务端内部提示词只用于上游请求，不向用户侧暴露；用户侧响应字段和正文均只展示用户请求模型。

## 文件变更清单

- `backend/app/services/proxy_service.py`
  - 统一增强 `claude-*` 系列模型身份提示。
  - 调整 Anthropic system 注入顺序，把内部身份保护提示追加到 system 末尾。
  - 新增用户可见正文身份泄露清洗函数。
  - 在 OpenAI、Anthropic、Responses 的响应模型字段重写函数中接入正文清洗。
- `md/plan-模型身份防泄露-20260615.md`
  - 记录方案、影响范围和实施步骤。

## 核心代码说明

1. `claude-*` 系列使用统一强身份提示，要求模型身份问题只回答用户请求模型 ID，并禁止提及 Codex、GPT、OpenAI、实际模型、上游模型、内部路由等信息。
2. Anthropic 请求的内部身份提示改为追加到 `system` 末尾，降低被 Claude Code / Codex CLI 自带 system 覆盖的概率。
3. `_sanitize_visible_model_identity_payload` 递归清洗用户可见响应文本字段，覆盖流式 chunk 和非流式完整响应。
4. 清洗规则只在检测到强泄露信号时触发，覆盖 `Codex`、`based on GPT`、`ChatGPT`、`OpenAI` 身份句、`实际模型/上游模型/底层模型` 等表达；普通讨论 GPT/OpenAI 的业务内容不做替换。
5. 清洗字段覆盖 `content/text/output_text/reasoning_content/thinking/summary` 等用户可见文本字段。
6. OpenAI / Anthropic 流式文本 delta 增加 256 字符尾部缓冲，降低 `Co` + `dex`、`Open` + `AI GPT-5` 这类跨 chunk 拆分泄露风险。
