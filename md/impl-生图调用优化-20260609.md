## 核心代码说明

### 图片有效性校验

`_parse_openai_image_response` 现在只接收通过以下校验的图片：

- 去除 `data:image/...;base64,` 前缀。
- 去除空白并补齐 padding。
- `base64.b64decode(..., validate=True)` 严格解码。
- 解码后必须匹配 PNG/JPEG/WebP/GIF 魔数。

如果所有图片均无效，后端返回失败，不扣图片积分。

### 实际有效图片数计费

图片请求仍按请求 `n` 做余额预检，避免上游成功后余额不足；但扣费时使用有效图片数重新计算：

```python
actual_charged_credits = model_multiplier * len(valid_images)
```

返回 payload 和 `RequestLog.image_count` 均使用实际有效图片数。

### CPA 图片接口 fallback

OpenAI 图片生成候选：

- `/v1/images/generations`
- `/v1/image/created`

OpenAI 图片编辑候选：

- `/v1/images/edits`
- `/v1/image/edit`

只有首选 URL 返回 404/405 时才 fallback，避免 5xx/超时情况下重复触发上游生成。

### Responses 生图

HTTP `/v1/responses` 现在支持：

- `model` 本身是图片模型。
- `tools` 或 `tool_choice` 包含 `type=image_generation`。

工具型生图必须显式指定图片模型，例如：

```json
{
  "model": "gpt-5.4-mini",
  "input": "生成一张 9:16 竖屏图",
  "tools": [{
    "type": "image_generation",
    "model": "gpt-image-2",
    "image_size": "2K",
    "aspect_ratio": "9:16"
  }]
}
```

websocket `/v1/responses` 本次不扩展生图，仍保留原有文本链路和安全拦截。

流式 Responses 生图成功时发送 `response.created`、`response.completed` 和 `[DONE]`；错误时发送 `response.created`、`event: error` 和 `[DONE]`，保证 OpenAI-compatible 客户端能结束等待。

## 接口文档

图片生成：

- `POST /v1/images/generations`
- `POST /v1/image/created`
- `POST /v1/responses`，使用 `image_generation` tool。

图片编辑：

- `POST /v1/images/edits`
- `POST /v1/image/edit`

推荐客户端 timeout：600 秒以上。
