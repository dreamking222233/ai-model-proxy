# impl-用户-套餐选购-20260622

## 任务概述

基于 `plan-用户-套餐选购-20260622.md` 实现用户前台在线购买套餐能力：

- 管理端套餐模板支持配置用户售价、代理拿货价、是否允许前台购买。
- 用户端 `/user/recharge` 新增“套餐选购”，套餐订单金额以服务端套餐售价为准。
- 套餐支付成功后复用 `SubscriptionService.activate_plan_subscription`，固定 `activation_mode=append`。
- 代理用户购买套餐后生成独立的 `agent_subscription_sale_record` 待返现核销记录。
- 管理端新增代理套餐销售页，支持查询、汇总、单条核销、批量核销。
- 代理端新增套餐销售页，支持查看销售额、待核销/已核销返现。
- 套餐订单不进入余额、图片积分、代理现金余额、推广充值奖励路径。

## 核心实现说明

### 套餐在线购买

用户从 `/api/user/payment/subscription-plans` 获取可购买套餐。后端只返回已启用、已上架且售价大于 0 的套餐；代理用户额外要求代理拿货价大于 0 且不高于售价。

创建套餐订单时：

- `recharge_type=subscription`
- 前端只传 `subscription_plan_id`
- 后端从 `subscription_plan.sale_price_cny` 读取支付金额
- 订单保存套餐模板快照，避免后续模板修改影响历史订单

### 支付成功入账

`PaymentService._apply_paid_order` 保持原有幂等结算表机制，套餐订单的 `asset_type=subscription`。

套餐订单入账只执行：

1. `SubscriptionService.activate_plan_subscription(..., activation_mode="append", auto_commit=False)`
2. 写回 `PaymentRechargeOrder.subscription_id`
3. 代理用户订单创建 `AgentSubscriptionSaleRecord`

不会执行：

- `_credit_user_balance`
- `_credit_user_image_credits`
- `_credit_agent_cash_balance`
- `PromotionService.apply_recharge_reward`

### 代理返现核销

代理套餐销售使用独立表 `agent_subscription_sale_record`，按 `order_no` 唯一，字段保存：

- 用户支付售价
- 代理拿货价
- 应返现金额
- 套餐快照
- 核销状态和核销人/时间/备注

管理端核销只更新销售记录状态，不自动调整代理现金余额，保留线下返现登记语义。
