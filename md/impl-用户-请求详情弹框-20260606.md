# 用户-请求详情弹框 Impl 20260606

## 任务概述

在 `user/balance` 页面增加用户可见的请求详情弹框。用户点击请求状态（成功、失败、超时等）后打开弹框查看请求详情，尤其是失败请求的错误信息。

## 文件变更清单

- `frontend/src/views/user/BalanceLog.vue`
  - 新增请求详情弹框模板。
  - 新增 `detailModalVisible`、`selectedRecord` 状态。
  - 新增 `detailModalTitle` 计算属性。
  - 状态列点击从展开行改为打开详情弹框。
  - 新增错误信息复制、弹框容器方法。
  - 补充弹框移动端布局样式。
- `backend/app/api/user/profile.py`
  - 用户日志接口增加 `status` 查询参数。
  - 用户日志查询内部获取完整字段后再转换为用户可见 DTO。
- `backend/app/services/log_service.py`
  - 新增用户可见请求日志白名单 DTO 构造方法。
  - 新增错误详情脱敏逻辑，隐藏实际模型、渠道名称、渠道 ID 和明显密钥片段。
- `backend/test_user_log_error_sanitization.py`
  - 覆盖 plain text、query-like、JSON string 三类错误详情脱敏场景。
- `md/plan-用户-请求详情弹框-20260606.md`
  - 记录实施方案。

## 核心代码说明

- 用户页继续使用现有 `/api/user/profile/usage-logs` 数据源。
- 用户日志接口最终只返回页面需要的白名单字段，不返回实际模型、渠道名称、渠道 ID、客户端 IP、用户 ID 等内部字段。
- `error_message` 在接口返回前按请求模型、实际模型和渠道名称做脱敏，弹框展示的是脱敏后的错误详情。
- 弹框只展示用户可见字段：请求 ID、请求模型 ID、请求类型、请求时间、响应耗时、用量、计费、缓存概览和错误详情。
- 弹框不展示实际模型、渠道名称、渠道 ID、客户端 IP、用户信息等内部字段。
- 表格行点击仍保留展开计费明细能力；状态标签点击用于打开请求详情弹框。
