# 用户专属倍率实施记录

## 任务概述

本次实现管理员按用户配置模型系列倍率：

- `admin/users` 可为单个用户维护专属倍率规则。
- 用户专属规则命中时覆盖全局模型分类倍率；未命中时回退全局规则；再未命中则使用默认 `x1`。
- `admin/price-adjustments` 展示所有用户专属规则，能看到用户、模型系列、倍率和生效状态。
- `admin/logs` 在“响应时间”和“计费”之间展示本次请求实际使用的倍率快照和来源。
- 文本、图片、视频计费链路均写入请求时的倍率快照，异步视频创建时冻结倍率并在完成扣费时复用。

## 核心实现说明

### 倍率解析优先级

`PriceAdjustmentService.resolve_adjustment(...)` 返回统一解析结果：

```text
用户专属规则 > 全局分类规则 > 默认 x1
```

文本最终价格倍率仍为：

```text
系统全局价格倍率 * 解析得到的分类/用户倍率 * Fast 倍率 * 长上下文倍率
```

媒体积分倍率为：

```text
基础图片/视频积分单价 * 解析得到的分类/用户倍率
```

### 日志快照

新增字段用于区分：

- 系统全局倍率：`global_price_multiplier_snapshot`
- 用户/全局分类倍率：`adjustment_price_multiplier_snapshot`
- 来源：`price_adjustment_source_snapshot`
- 规则 ID：`price_adjustment_rule_id_snapshot`

`admin/logs` 使用 `adjustment_price_multiplier_snapshot` 展示本次请求实际命中的用户倍率或全局倍率。

### 异步视频

创建视频任务时保存：

- `charged_credits`
- `model_multiplier`
- `adjustment_multiplier`
- `price_adjustment_source`
- `price_adjustment_rule_id`
- `billed`

任务完成后扣费读取创建时保存的快照，避免管理员修改规则后影响已创建任务。快照同时保存在内存和 `video_task_billing_snapshot` 表中，服务重启后仍可恢复路由和扣费参数。

异步视频查询接口在任务完成时即触发一次性扣费，并净化返回体中的上游直连地址，防止客户端只轮询查询接口并绕过 `/content` 代理下载造成漏扣费。
