# 用户端页面性能优化 Impl

## 任务概述

针对用户端页面大量 CSS/JS 特效导致性能压力的问题，按 Plan v2 对全局背景、用户端布局、用户页面、模型列表、统计页、充值页、聊天组件进行 UI/UX 性能优化。在保留科技感、卡片层次、状态反馈的前提下，移除持续动画、远程 Animate.css、列表过渡、大面积 blur 和 `transition: all`。

## 核心代码说明

1. 全局动效策略
   - 不再依赖远程 Animate.css。
   - 保留最小本地兼容类，避免大范围模板 class 删除。
   - 通过 `prefers-reduced-motion` 统一关闭动画、动画延迟和过渡。

2. 页面渲染成本
   - 全局背景不再持续移动或使用大范围 blur。
   - 用户端范围内目标扫描项已清零：
     - 远程 Animate.css import。
     - `transition-group`。
     - `backdrop-filter`。
     - `transition: all`。
     - `animation: ... infinite`。
     - `filter: blur(...)`。

3. 模型列表
   - 筛选结果直接由 CSS grid 渲染，避免过滤时批量入离场动画和 move 动画。
   - hover 由大幅 `translateY(-8px)` 降为轻量 `translateY(-2px)`。

4. 用量统计
   - resize 事件使用 120ms debounce，避免窗口变化期间连续触发 ECharts 重绘。
   - 销毁时取消 debounce 并释放实例引用，减少页面切换残留。
   - CountTo 时长从 1600ms 降为 800ms，减少动态效果偏好下为 0ms。
   - ECharts `animation` 根据系统减少动态效果偏好关闭。

5. 聊天生成状态
   - 取消 12 个粒子、核心 glow、圆环、图标 pulse、进度条滑动等持续动画。
  - 保留静态状态反馈，避免用户误以为生成状态丢失。

6. Review 迭代修复
   - reduced-motion 中增加 `animation: none` 和 `animation-delay: 0ms`，避免残留 inline delay 延迟显示内容。
   - Dashboard/UsageStats 的 CountTo 纳入减少动态效果控制。
   - UsageStats ECharts 显式设置 `animation: !reduceMotion`。
   - UsageStats resize 从 RAF 合并改为 120ms debounce。
   - 用户端 hover 位移和 scale 进一步收敛到轻量范围。
   - r2/r3 复审指出的 hover 大阴影已继续降级，最终 r4 复审通过。
