## 任务概述

修复 `user/chat` 页面生图成功后不显示图片的问题。

## 核心原因

前端聊天页先插入一条 `assistant` 占位消息：

- `kind = image_generating`
- `content = 正在生成图片...`

待接口成功返回后，再给这条消息补 `images` 字段。

但当前项目使用 Vue 2，直接给已存在对象追加新属性不是响应式更新，因此：

- 请求成功
- 图片缓存已写入
- 日志已成功记账
- 但消息组件没有感知到 `images` 新增，页面不显示图片

## 实施内容

### 1. 桌面聊天页

文件：

- `frontend/src/views/common/AiChat.vue`

修改：

- 图片生成占位消息初始化增加 `images: []`
- 图片编辑占位消息初始化增加 `images: []`
- 成功返回后将 `lastMsg.images = images` 改为 `self.$set(lastMsg, 'images', images)`

### 2. 移动聊天页

文件：

- `frontend/src/views/common/MobileChat.vue`

修改：

- 图片生成占位消息初始化增加 `images: []`
- 成功返回后将 `lastMsg.images = images` 改为 `this.$set(lastMsg, 'images', images)`

## 验证

执行：

```bash
cd frontend
npm run build
```

结果：

- 构建通过
- 仅有仓库既有体积 warning，无新增编译错误

## 影响说明

- 新生成的图片结果会在聊天页即时显示
- 该修复同时覆盖桌面和移动聊天页
- 不涉及后端接口、计费逻辑和数据库结构
