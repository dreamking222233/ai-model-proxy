# 联系方式门槛实施记录

## 任务概述

为直营用户端 QQ/微信联系方式增加付费门槛：用户通过 `user/recharge` 在线充值余额或购买套餐，累计成功付款人民币金额大于管理端配置门槛后，才展示平台 QQ 和微信。代理端逻辑保持不变。

## 文件变更清单

- `backend/app/api/user/profile.py`：新增直营联系方式门槛计算，按成功付款人民币累计值决定是否返回 QQ/微信。
- `backend/app/api/admin/system.py`：固定公告配置接口新增 `support_contact_threshold_cny` 门槛字段。
- `backend/app/api/public/site.py`：公开直营站点配置不返回 QQ/微信，避免未登录公开接口绕过。
- `backend/sql/upgrade_support_contact_threshold_20260705.sql`：生产升级脚本，幂等新增联系方式展示门槛配置。
- `frontend/src/views/admin/AnnouncementManage.vue`：管理端公告页新增门槛配置输入与说明。
- `frontend/src/views/user/Recharge.vue`：充值页支付说明/客服区域按门槛展示联系方式或提示。
- `frontend/src/views/user/Dashboard.vue`：用户首页官方支持卡片与登录公告弹窗按门槛展示联系方式或提示。
- `frontend/src/layout/UserLayout.vue`：公告抽屉按门槛展示联系方式或提示。
- `sql/initData.sql`、`backend/sql/initData.sql`：新增默认系统配置 `platform_support_contact_threshold_cny=100`。
- `md/plan-联系方式门槛-20260705.md`：记录实施方案。

## 核心代码说明

1. 门槛统计口径为 `payment_recharge_order`：
   - `status = 'paid'`
   - `agent_id IS NULL`
   - `site_scope = 'platform'`
   - `recharge_type IN ('balance', 'subscription')`
   - 聚合 `amount_cny`，累计值大于门槛才展示联系方式。
2. 门槛值来自 `system_config.platform_support_contact_threshold_cny`，不存在或格式异常时默认 100 元。
3. `GET /api/user/profile/site-config` 对未达标直营用户返回空的 `support_wechat/support_qq`，同时返回 `support_contact_visible=false` 和提示文案。
4. `GET /api/user/profile/announcements` 对直营固定公告使用同一脱敏逻辑；代理用户和代理站点配置不进入脱敏逻辑。
5. `GET /api/public/site-config` 没有登录用户上下文，直营站点统一隐藏 QQ/微信，代理站点仍返回代理自己的联系方式。
6. 前端不自行判断付款记录，只消费后端返回的 `support_contact_visible/support_contact_notice`，避免接口层仍泄露联系方式。
7. 生产升级 SQL 只在缺失时插入默认门槛，不覆盖管理员已经配置过的门槛金额。
