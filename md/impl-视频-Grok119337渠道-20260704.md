# impl-视频-Grok119337渠道-20260704

## 任务概述

新增 119337 Grok Video API 渠道类型，支持通过本系统统一视频接口创建、查询并代理下载 119337 上游异步视频任务。现有 CPA Grok 视频渠道逻辑保持不变。

## 文件变更清单

- `backend/app/services/channel_service.py`
  - 新增 `grok-video-119337` provider variant。
  - 新渠道默认关闭健康检查，并从图片能力列表排除。
- `backend/app/services/proxy_service.py`
  - 新增 119337 Grok Video 创建、查询、状态归一、结果 URL 提取和内容下载代理逻辑。
  - 将用户侧 `size` 转成上游 `aspect_ratio`，`resolution_name` 转成 `resolution`。
  - 上传参考图转为 data URL 后通过 `image_urls` 提交。
  - `grok-video-1.5` 强制单参考图；`grok-image-video` 支持最多 7 张参考图，多参考图最长按 10 秒提交。
  - 创建响应兼容顶层 `task_id/id` 与 `data.task_id/data.id`。
  - 119337 秒数归一到 `4/6/8/10/12/15`；多参考图 10 秒上限在计费前生效。
- `frontend/src/views/admin/ChannelManage.vue`
  - 管理端新增 `119337 Grok 视频` 渠道类型选项、标签、颜色和健康检查规则。
- `backend/sql/upgrade_grok_video_119337_channel_20260704.sql`
  - 新增 SQL 升级说明、provider variant 注释、模型元数据 upsert 和映射示例。
- `md/plan-视频-Grok119337渠道-20260704.md`
  - 记录实施方案。

## 核心代码说明

- 新渠道 `provider_variant = grok-video-119337`。
- 创建上游任务：
  - `POST {base_url}/v1/video/generations`
  - 请求体使用 JSON：`model/prompt/seconds/aspect_ratio/resolution/image_urls`。
- 查询上游任务：
  - `GET {base_url}/v1/video/generations/{task_id}`
  - `data.status=SUCCESS` 且存在 `data.result_url` 时归一为 `completed`。
  - `data.status=SUCCESS` 但缺少 `data.result_url` 时继续按处理中返回，避免提前结束。
  - `data.status=FAILURE` 时归一为 `failed`，错误信息取 `data.fail_reason`。
- 下载视频：
  - 成功后通过 `data.result_url` 代理流式下载；临时直链非上游 API host 时不携带渠道 API Key。
