## 任务概述

完成 Grok 视频生成模型接入，支持通过当前系统调用 grok2api 的视频任务接口。

新增对外接口：

- `POST /v1/videos`
- `POST /v1/created/video`
- `POST /v1/chat/completions`
- `GET /v1/videos/{video_id}`
- `GET /v1/videos/{video_id}/content`

支持能力：

- 文生视频
- 图生视频，`input_reference[]` 最多 7 张，超过返回 400
- `/v1/videos` 异步任务创建与任务查询
- `/v1/created/video` 创建任务后自动轮询，生成完成后返回任务信息和下载地址
- `/v1/chat/completions` 识别视频模型后走 Grok 文生视频链路，支持 `video_config` 与流式返回视频 URL
- 视频内容代理下载
- 成功创建任务后按媒体积分扣费并写入请求日志
- `user/chat` 页面可选择视频模型并发起文生视频、图生视频，结果在聊天页内展示
- `user/quickstart` 页面展示视频 API 调用方式、参数说明和查询/下载示例

## 核心代码说明

### 1. 视频路由

新增 `backend/app/api/proxy/video_proxy.py`：

- 创建接口解析 `multipart/form-data`
- 支持 `input_reference[]` 和兼容字段 `input_reference`
- 注册任务查询与视频内容下载接口

### 2. 视频上游转发

在 `ProxyService` 中新增视频处理链路：

- 校验 `model`、`prompt`、`seconds`、`size`、`resolution_name`、`preset`
- 要求统一模型 `model_type=video`
- 支持 `billing_type=image_credit/free`
- 通过 `model_channel_mapping` 查找 OpenAI 兼容渠道
- 将创建请求转发到上游 `/v1/videos`
- 将查询和下载请求转发到上游 `/v1/videos/{video_id}` 与 `/content`
- `/v1/created/video` 复用创建链路完成一次扣费，然后按任务状态自动轮询，直到 `completed/succeeded/success` 后返回
- `/v1/chat/completions` 在模型类型为 `video` 时绕过文本 token 计费，解析 `messages` 与 `video_config` 后转发上游 `/v1/chat/completions`
- 文生视频支持 `stream=true` 流式转发；非流式响应会补充 `usage`、`video_url` 等便于前端消费的字段

### 3. 任务路由与权限

创建任务成功后：

- 将上游返回的 `video_id` 写入 `request_log.upstream_session_id`
- 同时保存短期内存路由缓存，降低查询时数据库读取

查询或下载时：

- 先校验当前用户是否拥有对应 `video_id` 的成功请求日志
- 再按日志中的渠道转发到上游
- 避免无归属探测所有上游导致多租户任务泄露
- 视频内容下载使用流式代理，避免大文件一次性读入内存

### 4. 计费与日志

视频生成复用图片积分账户作为媒体积分：

- `billing_type=image_credit`：视频模型按 `image_credit_multiplier * seconds` 扣减，其中 `image_credit_multiplier` 表示每秒媒体积分单价
- `billing_type=free`：不扣减积分，仅记录日志
- 请求日志记录 `request_type=video_generation`
- 视频尺寸记录在 `image_size` 字段，扣减积分记录仍写入 `image_credit_record`

### 5. 管理端与文档

- 管理端模型类型新增“视频”
- 图片积分文案调整为媒体积分，兼容图片与视频
- 用户模型列表补充视频类型筛选和视频模型能力字段
- 用户快速开始新增 Chat Completions 文生视频、`/created/video` 图生视频、异步任务查询和下载示例
- 新增 `docs/grok-video-integration.md`

### 6. 用户聊天页

- `AiChat.vue` 增加视频模式工具栏，支持文生视频/图生视频切换
- 支持配置 `seconds`、`size`、`resolution_name`、`preset`
- 文生视频通过 `streamChat` 发送 `video_config`，从流式文本中提取视频 URL
- 图生视频通过 multipart 调用 `/v1/created/video`，系统等待完成后返回 `content_url`
- `ChatMessage.vue` 新增视频生成中状态与视频结果卡片，使用 `<video controls>` 展示结果

## Review 后迭代

针对自审结果已修复：

- `/v1/videos/{video_id}/content` 改为 `StreamingResponse` 流式代理，并转发 `content-length`、`content-disposition`。
- `backend/sql/initData.sql` 与 `sql/initData.sql` 均补齐 `grok-imagine-video` 初始化数据；`backend/sql/initData.sql` 同步补齐媒体积分相关字段。
- `seconds`、`size`、`resolution_name`、`preset` 在进入渠道循环前完成预校验，避免客户端参数错误触发多渠道请求。
- 参考图超过 7 张时返回 `TOO_MANY_VIDEO_REFERENCES`，不再静默截断。
- 视频计费改为按秒扣费，`grok-imagine-video` 默认 `0.500` 积分/秒。
- `/v1/created/video` 改为同步等待接口，创建上游任务后自动查询进度，完成后响应中返回 `content_url` 与 `final_status`；`/v1/videos` 保持异步语义。
- `/v1/chat/completions` 新增视频文生入口，支持 `video_config` 和流式返回。
- `user/chat` 页面新增视频模型处理，支持文生视频、图生视频和视频展示。
- `user/quickstart` 页面已同步视频生成调用方式和支持参数。
- 视频 Chat Completions 流式链路补充异常处理、渠道失败记录和失败请求日志。
- 视频 Chat Completions 跳过文本模型身份 prompt 注入，避免污染视频提示词。
- 聊天页图生视频保存 `contentUrl/videoCacheKey`，刷新后可用用户 API Key 重新拉取 blob 播放。
- 本地视频测试脚本改为从环境变量读取 base URL 与 API Key，避免硬编码密钥。
