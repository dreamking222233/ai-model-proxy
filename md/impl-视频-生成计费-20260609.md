# impl-视频-生成计费-20260609

## 任务概述

修复视频生成链路中“上游 2xx 但实际失败、无视频 URL 或视频地址返回错误内容时仍按成功计费”的问题。调整后，视频生成必须通过成功判定后才扣图片积分并写成功日志；失败路径只写失败日志，`image_credits_charged=0`。

  - OpenAI Chat Completions 流式视频分支：拦截上游 `[DONE]`，先校验完整文本中的视频 URL 和内容，成功计费后再发送 `[DONE]`；失败发送 error SSE 且不扣费。

## 核心代码说明

- `_validate_video_chat_response_or_raise`
  - 识别顶层 `error`、失败状态、`video_generation_failed`、`Asset upload returned 403` 等失败文本。
  - 必须提取到视频 URL，否则抛出 `OPENAI_VIDEO_GENERATION_FAILED`。
- `_validate_video_url_content_or_raise`
  - 对视频 URL 发起带 Range 的轻量 GET。
  - HTTP 非 2xx、空内容、JSON/text 错误内容均视为失败。
- `_non_stream_openai_video_chat_request`
  - 上游 chat completion 响应不再直接计费。
  - 必须先通过响应和视频内容校验，再写成功日志和扣积分。
- `_stream_openai_video_chat_request`
  - 拦截上游 `[DONE]`，先做响应文本和 URL 内容校验。
  - 校验失败写失败日志并向客户端发送 error SSE，不扣积分。
  - 校验成功后扣费并发送 `[DONE]`。
- `handle_video_request_and_wait`
  - 调用 `handle_video_request(..., bill_on_create=False)`，创建阶段不扣费。
  - `_wait_for_video_completion` 返回 completed 后先校验上游 `/content`，再幂等扣费。
- `handle_video_content`
  - 下载内容前校验上游 `/content`。
  - 校验通过后按创建阶段保存的计费上下文幂等扣费，重复下载不重复计费。
