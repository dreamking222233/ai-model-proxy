# 计费-上下文与Fast倍率 Impl

## 任务概述

新增文本请求长上下文计费倍率：

- 计费上下文 token = 输入 + 输出 + 缓存读取。
- 计费上下文超过 262,144 token 后，长上下文价格倍率为 2。
- Responses fast 模式继续使用 `service_tier=priority`，fast 价格倍率为 2。
- 同时开启 fast 且超过 262,144 token 时，综合倍率为 `基础价格倍率 * 2 * 2`。

## 核心代码说明

### 计费公式

余额真实扣费：

```text
total_cost = token_multiplier 后的计费 token * 模型单价 * price_multiplier * fast_multiplier * context_multiplier
```

套餐官方成本额度：

```text
quota_cost = raw token * 模型官方单价 * fast_multiplier * context_multiplier
```

`quota_cost` 继续不乘系统 `price_multiplier`，保持现有测试和套餐官方成本口径。

### 请求前预检查

- 余额模式：如果存在 `estimated_total_cost`，要求余额覆盖预计扣费；没有估算值时保留原最低余额阈值行为。
- 套餐成本额度：如果存在 `estimated_quota_cost`，要求当日剩余额度覆盖预计官方成本；没有估算值时保留原最低额度阈值行为。
- 套餐 Token 额度：如果存在 `estimated_total_tokens`，要求当日剩余额度覆盖预计 token；没有估算值时保留原已用额度达到上限才拦截的行为。
- 未设置输出上限时，预检查仍按估算输入 token 和长上下文倍率计算下限成本，避免长输入请求退回最低余额阈值。

### 长上下文判断

```text
context_tokens = raw_input_tokens + raw_output_tokens + raw_cache_read_input_tokens
context_multiplier = context_tokens > 262144 ? 2 : 1
```

边界条件：等于 `262144` 不加倍，`262145` 开始加倍。
