# 计费-请求前额度余额预检查 Impl

## 任务概述

完善文本请求的前置计费校验：用户请求发送到上游前，先判断套餐当日美元额度或余额是否大于 `0.1` 美元。满足阈值则放行请求，完成后按真实用量扣费或记录套餐额度；如果本次请求导致额度或余额归零、为负，下一次请求会被前置校验拦截。

## 文件变更清单

- `backend/app/services/subscription_service.py`
  - 新增文本请求最小美元阈值 `MIN_TEXT_REQUEST_USD_THRESHOLD = Decimal("0.1")`。
  - 成本口径套餐前置检查改为判断当日剩余额度是否大于 `0.1` 美元。
  - 套餐周期扣费更新改为直接累加本次实际用量，不再在后置阶段按阈值回退或改判。

- `backend/app/services/proxy_service.py`
  - 余额预检查改为余额大于 `0.1` 美元才允许发起文本请求。
  - 套餐额度不足时，余额大于 `0.1` 美元可走余额兜底，否则直接返回 `INSUFFICIENT_BALANCE`。
  - 后置记账不再把已放行请求改判为余额计费；一旦前置通过，后置只负责写入真实消费结果。

- `backend/app/test/test_subscription_compatibility.py`
  - 调整套餐额度前置测试为阈值规则。
  - 补充余额大于/等于阈值、套餐额度大于/等于阈值、请求后超额记账等覆盖。

- `md/plan-计费-请求前额度余额预检查-20260521.md`
  - 记录方案与实施步骤。

## 核心代码说明

1. `SubscriptionService.check_quota_before_request`
   - 成本口径套餐按 `quota_limit - used_amount > 0.1` 判断是否可继续请求。
   - Token 口径套餐保留原有 token 上限判断，因为它不是美元额度。

2. `SubscriptionService._apply_cycle_consumption_update`
   - 后置记账直接累加实际用量，不再在写回时重新按阈值回退。

3. `ProxyService._assert_text_request_allowed`
   - 没有可用套餐时，余额必须大于 `0.1` 美元。
   - 套餐额度不足时，余额必须大于 `0.1` 美元才能兜底走余额计费。
