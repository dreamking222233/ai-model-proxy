# 计费-长上下文模型开关 Impl

## 任务概述

将“超过 256k 上下文按 2 倍计费”从全局文本模型规则改为模型级开关。管理员可在 `admin/models` 新增或编辑模型时设置是否启用该规则；GPT 系列新增模型默认开启，Claude 等其它系列默认关闭。

## 文件变更清单

- `backend/app/models/model.py`
  - `UnifiedModel` 增加 `long_context_billing_enabled` 字段。
- `backend/app/schemas/model.py`
  - 模型创建、更新、详情 schema 增加 `long_context_billing_enabled`。
- `backend/app/services/model_service.py`
  - 模型列表/详情返回长上下文计费开关。
  - 新增模型时未显式传值则按模型系列默认：GPT=1，其它=0。
  - 更新模型时保存管理员提交的开关值。
- `backend/app/services/proxy_service.py`
  - 长上下文倍率计算改为读取模型开关。
  - 请求前预检查和实际扣费都传入 `unified_model`，保证倍率一致。
- `frontend/src/views/admin/ModelManage.vue`
  - 模型列表新增“长上下文”列。
  - 新增/编辑弹窗增加“长上下文 256k 后 2 倍计费”开关。
  - 新建 GPT 系列模型默认开启，其它系列默认关闭。
- `backend/sql/upgrade_model_long_context_billing_enabled_20260617.sql`
  - 新增幂等迁移，给 `unified_model` 添加字段，并将已有 GPT 文本/请求模型设为开启。
- `backend/sql/init.sql`、`backend/sql/initData.sql`、`sql/initData.sql`
  - 同步初始化表结构。
- `backend/test_long_context_billing.py`
  - 增加模型开关影响 256k 倍率的单元测试。
  - 增加 request 计费预检查回归测试，确保关闭开关时不会按 2 倍估算，开启开关时保留保守 2 倍估算。

## 核心代码说明

长上下文倍率当前逻辑：

```text
if unified_model.long_context_billing_enabled != 1:
    context_multiplier = 1
else:
    context_multiplier = context_tokens > 262144 ? 2 : 1
```

`context_tokens` 仍沿用原口径：

```text
raw_input_tokens + raw_output_tokens + raw_cache_read_input_tokens
```

关闭开关的模型仍会记录 `context_tokens_snapshot` 和阈值快照，但 `context_price_multiplier_snapshot` 为 `1`。按请求计费模型在输出上限未知时，只有开启长上下文计费开关才会使用保守 2 倍预检查。
