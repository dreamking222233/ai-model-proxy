## 任务概述

- 修复 `subscription_usage_cycle` 在高并发下因 `SELECT ... FOR UPDATE` 导致的锁等待与超时问题。
- 将订阅额度预检改为无锁读取。
- 将请求后额度记账改为数据库原子条件更新。
- 在本地记账事务边界增加瞬时数据库并发异常重试。

## 核心代码说明

### 1. 预检去锁化

- `SubscriptionService.check_quota_before_request()` 不再因为 `quota_precheck` 存在而对 `subscription_usage_cycle` 使用 `FOR UPDATE`。
- 现在仅执行周期读取/创建与快照判断，避免请求前的集中锁竞争。

### 2. 额度记账原子化

- 新增 `_apply_cycle_consumption_update()`：
  - 直接执行条件更新
  - `used_amount = used_amount + consumed_amount`
  - `request_count = request_count + 1`
  - 同步最新 `quota_metric/quota_limit`
- `consume_quota_after_request()` 改为：
  - 读取或创建周期记录
  - 执行原子更新
  - 成功后重新读取最新周期快照返回
  - 更新失败时重新判定是否真实超限

### 3. 事务级重试

- 新增 `SubscriptionService.is_retryable_concurrency_error()`，识别：
  - MySQL 1205 `lock wait timeout`
  - MySQL 1213 `deadlock found`
- `ProxyService._deduct_balance_and_log()` 改为包装 `_deduct_balance_and_log_once()`。
- 在整个本地记账事务层面进行最多 3 次有限重试，避免在已污染的 Session 内原地继续。
