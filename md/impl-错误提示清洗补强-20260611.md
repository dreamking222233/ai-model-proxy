# 错误提示清洗补强实施记录

## 任务概述

按用户要求放弃缓存策略与路由优化，仅保留最新重试和容错修复，并继续补强上游错误信息清洗，避免用户或代理端看到 `Upstream returned HTTP ...`、`providers=codex`、`auth_unavailable` 等上游原始返回。

## 文件变更

- `backend/app/services/log_service.py`
  - 新增用户可见日志的原始上游错误启发式识别。
  - 对上游 HTTP 错误、结构化 error payload、provider/channel 元数据统一返回 `调用失败，渠道异常，请稍后重试`。
  - 新增代理端可见日志 DTO，保留用户身份字段，但隐藏真实模型、渠道、客户端 IP 和上游错误原文。
- `backend/app/api/agent/log.py`
  - 代理端请求日志列表改为返回清洗后的代理可见 DTO。
- `backend/test_user_log_error_sanitization.py`
  - 增加 `auth_unavailable/providers=codex`、非 JSON HTTP 403、中文上游错误以及代理端 DTO 的回归测试。

## 验证

```bash
env PYTHONPATH=backend python -m py_compile backend/app/services/log_service.py backend/app/api/agent/log.py backend/test_user_log_error_sanitization.py backend/test_proxy_retry_error_sanitization.py
env PYTHONPATH=backend python -m unittest backend/test_user_log_error_sanitization.py backend/test_proxy_retry_error_sanitization.py
```
