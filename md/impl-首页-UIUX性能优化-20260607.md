# 首页-UIUX性能优化 Impl

## 任务概述

针对 `frontend/src/views/Home.vue` 首页打开卡顿问题，在保留现有科技背景、Hero、3D 大模型星系、模型列表等视觉效果的前提下，降低首屏渲染成本、持续动画 CPU 占用和页面切换后的资源残留。

## 文件变更清单

- `frontend/src/views/Home.vue`
  - 模型展示数据改为接口返回后一次性预计算。
  - Canvas 粒子动画改为自适应粒子数量、15fps 绘制节流、页面不可见暂停。
  - 补齐 resize、visibilitychange、timeout、requestAnimationFrame 生命周期清理。
  - 模型列表去除 `transition-group` 批量动画，减少过滤切换时的 DOM 重排。
  - 优化星系节点、卡片 hover、下方区块延迟渲染和移动端响应式尺寸。
  - 将大模型星系改为圆形旋转并收缩尺寸，同时增加轨道容器安全高度，避免提示文案与底部模型节点重叠。
  - 统一接口模型与兜底模型的前台展示名，将 `GPT-5.5 Ultra`、`GPT-5.4 Omni` 分别展示为 `GPT-5.5`、`GPT-5.4`。
- `frontend/src/utils/brand-logos.js`
  - OpenAI/Grok logo 改为从 `src/assets/provider-icons` 导入，避免硬编码构建哈希路径导致 dev server 断图。
- `md/plan-首页-UIUX性能优化-20260607.md`
  - 记录优化方案与实施步骤。

## 核心代码说明

1. 新增 `normalizeModels`、`selectGalaxyModels`、`withNodeStyles` 等纯函数，将品牌 SVG、品牌 class、价格文案、卡片 id、星系 transform 预先写入视图模型，模板不再反复调用 `getBrandSvg/getBrandClass/getPriceDisplay/getNodeStyle`。
2. `fetchData` 使用 `Promise.allSettled` 并行拉取站点配置和公开模型列表，页面初始直接渲染兜底模型，接口失败时保持可用展示。
3. `initCanvasParticles` 根据视口面积控制粒子数量，绘制频率限制到 15fps，并在 `document.hidden` 时取消 RAF，恢复可见时再继续。
4. 点击模型定位时统一管理 `flashTimer`，避免重复点击产生多个定时器；悬停相同模型时避免重复响应式赋值。
5. CSS 中降低大范围 blur 成本，限定动画 transition 属性，增加 `content-visibility`、`will-change`、`backface-visibility` 和移动端星系半径适配。
6. 根据 Review 迭代：
   - Canvas 调度改为 `setTimeout + requestAnimationFrame`，避免未到绘制间隔时仍按屏幕刷新率唤醒。
   - resize 时重建粒子池，保持不同视口下的粒子密度和分布。
   - 有真实模型但类型不在 `chat/image/video` 时，星系展示真实模型前 15 个，不再回退兜底假数据。
   - 监听 `prefers-reduced-motion` 变化，运行时切换系统动效偏好会同步停止或恢复 canvas。
