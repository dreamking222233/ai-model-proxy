# 套餐日志与周期修复

日期：20260510

## 适用场景

- `admin/subscription`、`agent/subscription` 页面里，旧套餐当前周期仍显示 `Token` 或 `0.00`。
- `admin/logs`、`agent/logs`、`user/balance` 里，实际成功请求被显示成 `失败 / FREE`。
- 需要在正式环境对指定用户做一次性修复，不想改全库历史数据。

## 说明

- 代码已支持自动修正当前活跃套餐的周期口径。
- 如果你想让正式环境立刻恢复正确显示，可以先部署代码，再按下面 SQL 处理指定用户。
- 下面所有 SQL 都建议先备份，再执行。

## 1. 修复某个用户的当日套餐周期

先查出用户对应的 `subscription_id`：

```sql
SELECT id, user_id, plan_code_snapshot, plan_name, start_time, end_time, status
FROM user_subscription
WHERE user_id = 374
ORDER BY id DESC;
```

再修复当天周期（把 `374` 和 `2026-05-10` 换成目标用户和周期日期）：

```sql
UPDATE subscription_usage_cycle c
JOIN user_subscription s ON s.id = c.subscription_id
LEFT JOIN (
    SELECT
        cr.subscription_id,
        COUNT(*) AS request_count,
        COALESCE(SUM(cr.total_cost), 0) AS used_amount,
        MAX(cr.id) AS last_record_id
    FROM consumption_record cr
    WHERE cr.subscription_id = (SELECT id FROM user_subscription WHERE user_id = 374 ORDER BY id DESC LIMIT 1)
      AND cr.created_at >= '2026-05-10 00:00:00'
      AND cr.created_at < '2026-05-11 00:00:00'
      AND cr.model_name IS NOT NULL
    GROUP BY cr.subscription_id
) x ON x.subscription_id = c.subscription_id
LEFT JOIN consumption_record lr ON lr.id = x.last_record_id
SET
    c.quota_metric = 'cost_usd',
    c.quota_limit = 120.000000,
    c.used_amount = COALESCE(x.used_amount, 0),
    c.request_count = COALESCE(x.request_count, 0),
    c.last_request_id = lr.request_id
WHERE s.user_id = 374
  AND c.cycle_date = '2026-05-10';
```

> 说明：
> - `quota_limit` 按套餐实际规则填写，存量无限月卡通常是 `120`，新开通常是 `100`。
> - `daily-unlimited / weekly-unlimited / monthly-unlimited` 都是美元口径。

## 2. 修复“请求成功但本地记账失败”的日志显示

这类记录在日志里原本会被写成 `error`，但实际请求已经成功返回。

建议把它们统一改成 `success`，仅保留错误信息作为备注：

```sql
UPDATE request_log
SET status = 'success'
WHERE status IN ('error', 'failed')
  AND error_message LIKE '本地计费或记账失败：%'
  AND total_tokens > 0;
```

## 3. 验证

```sql
SELECT id, request_id, status, total_tokens, error_message
FROM request_log
WHERE user_id = 374
ORDER BY id DESC
LIMIT 20;

SELECT cycle_date, quota_metric, quota_limit, used_amount, request_count, last_request_id
FROM subscription_usage_cycle
WHERE subscription_id = (SELECT id FROM user_subscription WHERE user_id = 374 ORDER BY id DESC LIMIT 1)
ORDER BY cycle_date DESC;
```
