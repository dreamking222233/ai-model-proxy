# plan-CPA视频渠道接入-20260627

## 用户原始需求

新增 CPA 类型的渠道视频接入方式，同时确保返回给用户的内容和 JSON 不改变。

## 技术方案设计

- 在渠道 `provider_variant` 中新增 CPA Grok 视频变体，优先不新增新的 `protocol_type`，继续使用 `openai` 协议，避免影响现有模型/渠道选择逻辑。
- 对 CPA Grok 视频渠道单独适配上游请求：创建走 CPA 原生 `/v1/videos/generations` JSON 接口，查询走 `/v1/videos/{id}`。
- 对外仍保持当前系统的 `/v1/videos`、`/v1/created/video`、`/v1/videos/{id}`、`/v1/videos/{id}/content` 响应结构；创建响应仍包含 `id`，同步接口仍返回 `content_url` 和 `retrieve_url`。
- 图生视频参考图继续接受用户 multipart 上传，但转发 CPA 时转换成 data URL，避免改变用户调用方式。
- 计费仍沿用现有媒体积分逻辑：创建阶段不扣费，完成并校验视频内容后扣费。
- CPA 时长按 1-15 秒处理；非 CPA 保持原有 6/10/12/16/20 限制，避免破坏旧渠道。
