# 聊天图片二次操作方案（2026-04-23）

## 用户原始需求
- 对聊天页中已经生成出来的图片，新增二次操作入口。
- 期望在图片旁边出现操作按钮，例如：
  - 编辑
  - 重新生成

## 技术方案设计
1. 图片结果卡片扩展
- 在 `ChatMessage.vue` 的 `image_result` 消息区增加“二次操作”按钮组。
- 基于当前系统 `n=1` 的图片结果，默认对当前消息中的首张结果图进行操作。

2. 编辑这张图
- 点击后触发父组件事件。
- 父组件读取该图片缓存：
  - 若当前模型支持图片编辑，则直接切换到编辑模式
  - 若当前模型不支持，则尝试切换到可编辑的图片模型
- 将选中的结果图回填到编辑上传区：
  - 生成浏览器预览
  - 转换为 `File` 供后续 `/v1/image/edit` 提交
- 同时回填原提示词到输入框，便于继续修改。

3. 重新生成
- 点击后切回图片生成模式。
- 回填原消息中的提示词到输入框。
- 尽量切换回该消息原先使用的模型，方便直接再次发送。

4. 用户体验补充
- 完成回填后给出轻量提示。
- 自动聚焦输入框，降低再次操作成本。
