# 上游失败重试优化 Plan

## 用户原始需求

优化重试机制：只要用户发送请求后已经调用上游，遇到上游失败均进行重试；但用户自身问题不重试，例如余额不足、上下文超出、系统本地直接拒绝的请求。

## 技术方案设计

1. 保留本系统本地预检逻辑不变：余额不足、上下文超出、请求长度超限等在调用上游前直接返回，不进入上游重试。
2. 扩展上游 HTTP 状态码重试判断：已调用上游后，除明显用户请求问题外，非 2xx 上游失败默认可重试。
3. 对明显用户请求问题设置排除规则，例如 `400`、`413`、`422`，避免无意义重试。
4. 对 Responses SSE 首包 `type=error` 使用同类判断：在没有向用户输出任何内容前可重试；一旦已经输出内容，则不重试，避免重复或破坏流式响应。
5. 增加单元测试覆盖：通用上游失败可重试、用户请求错误不重试、Responses SSE 上游错误重试、Responses SSE 请求错误不重试。
