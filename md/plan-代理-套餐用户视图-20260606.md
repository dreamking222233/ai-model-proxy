## 用户原始需求

在当前代理端 `agent/subscription` 页面，参考 `admin/subscription` 页面添加一个“套餐用户”的显示，可以看到当前代理下用户的套餐情况，比如剩余多少天等信息，方便代理对用户进行催费。

## 技术方案设计

当前管理端已经有完整的“套餐用户”查询和展示链路，代理端已有套餐发放和发放记录，但缺少“当前代理名下套餐用户列表”。

本次实现采用“后端新增代理范围接口 + 前端复用管理端展示模式”的方案：

1. 后端在 `agent/subscription` API 增加“代理套餐用户列表”接口
2. 服务层复用订阅数据查询逻辑，按 `current_user.agent_id` 过滤用户
3. 返回字段对齐管理端所需核心信息：
   - 用户 ID / 用户名
   - 套餐名称 / 套餐类型
   - 开始时间 / 到期时间
   - 剩余天数
   - 套餐状态
4. 前端在 `agent/subscription` 页面增加“套餐用户”标签页
5. 视图风格参考 `admin/subscription`，但仅保留代理催费需要的核心字段和操作

## 实施步骤概要

1. 阅读管理端套餐用户接口与页面结构
2. 在服务层补代理范围的套餐用户列表查询
3. 在代理 API 暴露新接口
4. 在代理前端 API 增加调用方法
5. 在 `agent/subscription` 增加“套餐用户”标签页和表格
6. 补充剩余天数/状态显示逻辑
7. 运行定向校验
