# 代理套餐在线充值与套餐排序 Plan

## 用户原始需求

1. `agent/system` 页面新增是否开启对应站点的套餐在线充值。
2. `admin/subscription` 页面可以控制套餐排序，用于 `user/recharge` 页面套餐展示顺序。

## 技术方案设计

- 在 `agent` 表新增 `subscription_online_recharge_enabled` 字段，独立控制代理站点的套餐在线购买入口。
- 公共站点配置接口返回 `subscription_online_recharge_enabled`，代理端系统配置接口允许代理保存该字段。
- 用户充值页读取站点配置：
  - 总在线充值关闭时，保持整页不可用。
  - 套餐在线充值关闭时，只隐藏/禁用套餐购买入口，按量充值仍可用。
- 后端创建充值订单时对 `recharge_type=subscription` 增加站点开关校验，避免绕过前端。
- 管理端套餐模板表和编辑弹窗增加 `sort_order` 字段；后端已有 `sort_order asc, id asc` 排序，`user/recharge` 会按该顺序展示。

## 实施步骤概要

1. 新增代理站点套餐在线充值字段、schema、序列化和公开站点配置返回。
2. 后端套餐下单接口校验站点套餐在线充值开关。
3. 代理系统配置页新增“套餐在线充值”开关并保存。
4. 用户充值页按开关显示或禁用套餐付费入口。
5. 管理员套餐模板页增加排序列和排序输入。
6. 更新本地数据库并执行后端测试、接口验证、前端构建。

## 部署前置

- 本功能依赖用户套餐在线购买的数据结构，部署时必须先执行 `backend/sql/upgrade_user_subscription_purchase_20260622.sql`，再执行 `backend/sql/upgrade_agent_subscription_online_recharge_20260622.sql`。
