# 使用排行改为美刀

## 用户原始需求
- 将 `admin/ranking` 和 `user/ranking` 页面中的“使用排行”从按 Token 排行改为按美元消费排行。

## 技术方案设计
1. 后端保留现有排行接口入口，调整排行聚合逻辑：
   - 从 `RequestLog.total_tokens` 排序改为基于 `ConsumptionRecord.total_cost` 汇总排序。
   - 返回字段补充 `total_cost`，保留原有请求次数、最近使用时间等辅助信息。
2. 前端复用现有排行页组件：
   - 将文案从 `Token`/`Tokens` 改为美元消费。
   - 将榜单主数值从 `total_tokens` 切换为 `total_cost`。
   - 保留请求次数等辅助信息。
3. 管理端 `admin/ranking` 因为复用用户排行页组件，会随组件改动同步生效。

## 涉及文件清单
- `backend/app/services/log_service.py`
- `frontend/src/views/user/UsageRanking.vue`
- `md/impl-使用排行改为美刀-20260510.md`

## 实施步骤概要
1. 修改后端排行查询逻辑，按 `ConsumptionRecord.total_cost` 聚合并返回金额字段。
2. 修改排行页展示文案和数值格式，切换为美元显示。
3. 运行后端语法校验和前端构建验证。
4. 记录实施结果到 impl 文档。
