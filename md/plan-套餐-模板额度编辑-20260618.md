# 套餐-模板额度编辑 Plan

## 用户原始需求

`admin/subscription` 页面编辑套餐额度后没有反应，需要检查并修复。

## 技术方案设计

当前套餐模板列表和编辑弹窗优先使用 `resolved_quota_metric/resolved_quota_value`。后端对内置畅享套餐编码存在兼容规则，会把默认 Token 额度解析为固定美元额度，导致模板原始额度即使保存成功，页面展示和后续发放仍可能看起来没有变化。

修复方向：

1. 管理端“套餐模板”列表和编辑弹窗使用模板自身的 `quota_metric/quota_value`，避免管理配置被运行时解析值覆盖。
2. 后端兼容规则只对未改过的内置畅享默认配置生效；管理员修改额度口径或额度值后，后续发放按模板保存值生成套餐快照。
3. 保留已有用户套餐记录的兼容解析，避免历史记录行为突然变化。
4. 增加单测覆盖“默认内置畅享仍兼容”和“修改后模板按保存值生效”。
