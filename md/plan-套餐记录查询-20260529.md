# 套餐记录查询 Plan

## 用户原始需求

在 `admin/subscription` 页面新增一个可以根据用户名或 ID 查询对应用户套餐记录的能力。

## 技术方案设计

- 后端扩展管理员套餐记录列表接口 `/api/admin/subscription/list`：
  - 新增 `keyword` 查询参数。
  - 数字关键字按用户 ID 精确匹配，同时兼容用户名/邮箱包含该数字。
  - 非数字关键字按用户名、邮箱模糊匹配。
- 服务层 `SubscriptionService.list_all_subscriptions` 增加 keyword 过滤，继续复用现有分页、状态筛选、套餐序列化和当前周期计算逻辑。
- 前端 `SubscriptionManage.vue` 的“套餐记录”工具栏增加搜索框和查询/重置按钮。
- 前端 API `listAllSubscriptions` 继续透传 params，无需新增 API 方法。
- 增加后端单元测试覆盖用户名与 ID 查询参数传递和过滤构造。

## 实施步骤概要

1. 后端接口新增 `keyword` 参数。
2. 服务层列表查询增加用户 ID/用户名/邮箱过滤。
3. 前端套餐记录 toolbar 增加搜索输入、查询、重置。
4. 前端请求列表时携带 `keyword`。
5. 增加并运行相关测试、编译检查。
