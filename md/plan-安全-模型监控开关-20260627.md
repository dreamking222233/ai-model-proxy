# 安全-模型监控开关 Plan

## 用户原始需求

当前安全风控会对用户请求做违规检测、拦截、快照和管理端记录。现在需要按模型控制是否启用安全监控：

- 管理端 `admin/models` 已能维护模型系列，如 `claude`、`gpt`、`grok`。
- 需要在管理单个模型时设置是否添加安全监控。
- 开启安全监控的模型：继续走现有安全风控逻辑，危险请求会拦截、记录、管理端可查看。
- 关闭安全监控的模型：用户使用该模型时不做任何安全风控处理。
- 目标策略示例：限制 `claude` 和 `gpt`，放开 `grok`。

## 技术方案设计

### 数据模型

在 `unified_model` 增加字段：

- `security_monitor_enabled` tinyint not null default 0
- 含义：是否对该模型启用现有安全风控链路。

初始化和升级策略：

- 新字段默认 `0`，避免其他系列模型被意外纳入风控。
- 升级脚本对已有数据按系列和模型名前缀回填：`gpt`、`claude`、`gpt/o1/o3/o4/claude` 前缀设置为 `1`，其他设置为 `0`。
- 新建模型时，如果前端未传该字段，后端按系列默认：`gpt`、`claude` 为 `1`，其他为 `0`。

### 管理端

在 `frontend/src/views/admin/ModelManage.vue`：

- 表格增加“安全监控”列，展示开启/关闭。
- 新增/编辑模型弹窗增加开关。
- 新建模型时根据模型系列自动同步默认值：`gpt`/`claude` 开启，`grok` 关闭；用户手动改动后不再自动覆盖。

### 后端模型管理

涉及：

- `backend/app/models/model.py`
- `backend/app/schemas/model.py`
- `backend/app/services/model_service.py`

处理：

- ORM、Pydantic schema、返回字典都包含 `security_monitor_enabled`。
- create/update 支持读写该字段。

### 代理入口

涉及 `backend/app/services/proxy_service.py`。

新增 helper：

- `_is_security_monitor_enabled(unified_model)`
- `_maybe_create_security_snapshot(...)`
- `_maybe_scan_security_request_or_raise(...)`
- `_maybe_inject_security_prompt(...)`

调整入口：

- OpenAI Chat、Anthropic、Responses WebSocket、图片、视频、图片编辑、视频 Chat Completions：先解析目标 `UnifiedModel`，再按开关决定是否执行安全快照、扫描、提示词注入和输出扫描关联。
- Responses HTTP 文本请求：先解析模型；图片型 Responses 分流到图片入口，由图片入口按目标图片模型处理。

关闭安全监控的模型必须跳过：

- `security_request_snapshot` 创建
- `SecurityDetectionService.ensure_allowed_or_raise`
- 安全提示词注入
- 输出 `[MIS_RISK_REPORT ...]` 捕获记录

保留：

- 现有模型身份提示词不受该开关影响。
- 余额、套餐、上下文、渠道、计费等业务校验不受该开关影响。
