# 安全-登录与CORS加固 Plan

## 用户原始需求

修复当前系统登录凭证、跨域、注册登录防护、Source Map、响应头、nginx 版本暴露、邮箱验证、JWT 无法单独吊销等安全问题，避免不同域名部署时错误加载 `xiaoleai.team` 数据。

## 技术方案设计

1. CORS 收敛
   - 后端继续使用白名单 + 动态代理域名 CORS。
   - nginx 不再反射 `$http_origin`，预检请求回源给 FastAPI 处理。
   - 保留 `Access-Control-Allow-Credentials: true` 仅给后端允许的 Origin。

2. 前端 API 域名环境化
   - 生产环境不再硬编码 `https://api.xiaoleai.team`。
   - 统一读取 `VUE_APP_BASE_API`，未配置时使用同源 API，避免复制部署误连平台 API。

3. Source Map 关闭与访问阻断
   - Vue CLI 生产构建关闭 `productionSourceMap`。
   - nginx 前台模板/现有配置增加 `.map` 拒绝访问。

4. 安全响应头与 nginx 版本
   - nginx 配置增加 HSTS、X-Frame-Options、X-Content-Type-Options、Referrer-Policy、Permissions-Policy、基础 CSP。
   - 在 nginx 配置中增加 `server_tokens off;` 提示，实际生产仍建议放到全局 `http` 块。

5. 注册/登录/忘记密码限流
   - 新增认证限流服务，优先 Redis，Redis 不可用时使用进程内存窗口。
   - 按 IP + 用户名/邮箱组合限流，覆盖登录、注册、忘记密码验证、重置、邮箱验证码发送。

6. JWT 可吊销增强
   - JWT 增加 `iat`、`jti`、`token_version`。
   - `sys_user` 增加 `token_version` 字段，校验时比对；改密码、重置密码、禁用用户时递增，让旧 token 失效。
   - 提供 SQL 升级脚本。

7. 注册邮箱验证
   - 新增邮箱验证码发送接口。
   - 注册请求增加 `email_code`。
   - 使用 Redis/内存保存验证码哈希和过期时间；SMTP 未配置时拒绝发送，避免生产环境绕过。
   - 配置项可控制是否强制邮箱验证，生产默认开启。
