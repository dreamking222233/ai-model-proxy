# 安全风控-序列化会话误判 Plan

## 用户原始需求

- 放开用户 `DDD` 最近因 `keygen`/“后门”历史上下文造成的安全风控误判，并更新生产服务器。

## 技术方案

1. 不信任客户端序列化 JSON 中的 `role`，继续扫描外层用户消息的完整文本，避免通过伪造 `assistant` 角色绕过风控。
2. 将 `keygen` 从无条件高风险词调整为上下文规则：`ssh-keygen` 等正常 SSH 工具语境不命中，与制作、破解、授权绕过等语境组合时仍拦截。
3. 为“后门”增加明确否定语境识别，“没发现/未发现后门”等审计结论不拦截，“植入后门”等攻击性语境继续拦截。
4. 新增回归测试，覆盖 `DDD` 的 `ssh-keygen`/否定式安全审计语境、真实 `keygen` 滥用语境，以及伪造 `assistant` 角色不得绕过扫描。
5. 恢复被后续视频功能提交覆盖的请求扫描 fail-closed 逻辑，扫描未知异常时固定返回 `503 SECURITY_DETECTION_UNAVAILABLE`。
