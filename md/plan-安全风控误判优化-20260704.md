# 安全风控误判优化 Plan

## 用户原始需求

用户 `1684567998@qq.com` 使用过程中反馈 403：`请求可能涉及违规或高风险内容，已被安全策略拦截。你可以改为询问合法合规的学习、防护或排查建议。`

生产库排查显示最近误判主要包括：
- 技术上下文中的 `prompt injection` 被直接判定为 `prompt_jailbreak`。
- 普通中文词组 `阶段性交付` 被 `性交` 子串命中。

目标是在不关闭风控、不放开真实违规请求的前提下，降低正常研发/文档场景误伤。

## 技术方案设计

1. 保留直接高风险词策略，但将英文 `prompt injection` 从直接关键词列表移出，改成上下文匹配：
   - 命中恶意/绕过意图词时继续阻断。
   - 命中修复、防护、漏洞分析、测试、代码审查等研发安全上下文时放行。
2. 调整 `性交` 的歧义正则，排除 `阶段性交付` 等常见普通中文组合。
3. 增加回归测试覆盖线上误判样例，同时保留真实绕过提示词阻断。

## 涉及文件清单

- `backend/app/services/security_detection_service.py`
- `backend/app/test/test_security_detection_false_positive.py`

3. 优化中文歧义词正则。
4. 新增回归测试。
5. 运行相关测试与 Python 编译检查。
