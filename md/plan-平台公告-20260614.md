# 平台公告功能实施方案

## 用户原始需求

管理端新增公告管理页面，可以编辑当前直营客户登录后看到的开屏公告，也可以发布新公告。固定开屏公告展示后，若存在额外新公告，用户点击“好的，去体验”后继续弹出第二条公告。用户端顶部导航栏头像左侧增加公告图标，点击后可查看当前平台公告内容。

## 技术方案设计

1. 保留现有 `system_config` 中的 `platform_announcement_title`、`platform_announcement_content`、`platform_support_wechat`、`platform_support_qq` 作为直营站点固定开屏公告配置来源，避免破坏 `public/site-config`、代理站点配置和用户 Dashboard 既有逻辑。
2. 新增 `platform_announcement` 表存储额外公告，支持标题、内容、状态、是否开屏展示、排序、发布时间、创建/更新时间。
3. 后端新增管理接口：
   - `GET /api/admin/system/announcements/config` 获取固定公告配置。
   - `PUT /api/admin/system/announcements/config` 保存固定公告配置。
   - `GET /api/admin/system/announcements` 分页查询额外公告。
   - `POST /api/admin/system/announcements` 新建公告。
   - `PUT /api/admin/system/announcements/{id}` 编辑公告。
   - `DELETE /api/admin/system/announcements/{id}` 删除公告。
   - `PUT /api/admin/system/announcements/{id}/status` 发布/下线公告。
4. 后端新增用户接口 `GET /api/user/profile/announcements`，按当前站点返回公告列表：
   - 直营站点：管理端固定开屏公告 + 管理端已发布额外公告。
   - 代理站点：代理端自定义固定开屏公告 + 管理端已发布额外公告。
   - 管理端固定开屏公告只适用于直营用户；管理端额外公告面向全部用户，包括代理端用户。
5. 用户端将 Dashboard 内单次弹窗改为公告队列弹窗，sessionStorage 按公告 ID 记录当次会话已展示，避免重复弹出。
6. 用户布局 Header 头像左侧增加公告图标，点击打开抽屉/弹窗展示当前公告列表。
7. 管理端新增 `/admin/announcements` 页面和菜单，提供固定公告表单与额外公告列表、发布、编辑、删除能力。

## 实施步骤概要

1. 新增公告 ORM 模型和 SQL 升级脚本。
2. 后端实现管理端固定公告配置与额外公告 CRUD 接口。
3. 后端实现用户端当前站点公告列表接口。
4. 前端新增管理端公告 API、用户端公告 API。
5. 新增管理端公告管理页面，并接入路由和菜单。
6. 用户端 Header 增加公告图标与公告查看抽屉。
7. 改造 Dashboard 登录后弹窗为顺序公告队列。
8. 执行前后端基础验证并记录实施结果。
