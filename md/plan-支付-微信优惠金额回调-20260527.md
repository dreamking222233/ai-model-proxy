# 支付-微信优惠金额回调 Plan

## 用户原始需求

微信支付订单在用户使用微信优惠后，系统提示“微信支付回调金额与本地订单不一致”，但商户后台已收到款。需要修复微信优惠支付场景，确保原价支付和优惠支付都能正常回调入账；同时在 `user/recharge` 页面右侧增加提醒：支付后未到账请联系微信、QQ，代理端展示代理设置的微信和 QQ。

## 技术方案设计

1. 微信支付下单仍按本地订单金额传 `amount.total`，不调整下单逻辑。
2. 微信回调/查单入账校验时，优先使用微信交易对象中的 `amount.total` 与本地 `amount_cny` 比较。
3. `amount.payer_total` 仅代表用户优惠后实付金额，不能作为商户订单金额校验依据。
4. 如果上游极端情况下没有 `amount.total`，保留 `payer_total` 兜底，维持旧 payload 兼容。
5. 补充单测：`amount.total=1000`、`amount.payer_total=604` 时，10 元订单应入账成功。
6. 充值页右侧支付说明区域新增未到账提醒，联系方式使用现有 `siteConfig.support_wechat/support_qq`。
7. 代理站点联系方式由现有站点配置接口按域名返回，无需新增接口。
