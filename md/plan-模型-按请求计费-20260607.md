# 模型-按请求计费 Plan

## 用户原始需求

当前系统模型请求计费按请求 token 和每百万 token 价格计算。需要新增模型支持按请求次数计费，在 `admin/models` 新增/编辑模型时可选择按 Token 或按请求次数计费，例如每次请求固定计费 1 美元。原有 token 超过 25w 时 x2 的长上下文计费逻辑，按次数计费也需要支持。

## 技术方案设计

- 在 `unified_model` 增加 `request_price` 字段，表示按请求计费时每次请求的美元价格。
- `billing_type` 新增 `request` 类型，保持现有 `token/image_credit/free` 不变。
- 文本请求计费链路在 `ProxyService._deduct_balance_and_log_once` 判断模型计费类型：
  - `token`：沿用输入/输出 token 单价公式。
  - `request`：按 `request_price * 系统价格倍率 * fast倍率 * 长上下文倍率` 计算总价。
  - `free`：计费为 0。
- 长上下文倍率继续复用现有 `context_tokens_snapshot > _LONG_CONTEXT_TOKEN_THRESHOLD` 判断。
- 预检额度估算 `_build_text_quota_precheck` 同步支持按次价格，避免余额/套餐预检漏算。
- 请求日志和消费记录增加 `request_price_snapshot`，用于保留当次按次价格快照。
- 管理端模型列表和表单增加按次计费选择和每次请求价格输入。
- 用户端模型列表和公开首页返回/展示按次计费价格。
- 新增 SQL 升级脚本，并同步 `sql/initData.sql` 与 `backend/sql/initData.sql`。

## 实施步骤概要

1. 扩展数据库模型、Pydantic schema、ModelService 序列化和保存字段。
2. 扩展计费服务，增加按请求计费成本计算、预检估算和日志快照。
3. 扩展用户/公开接口返回 `request_price`。
4. 更新管理端模型页面，支持选择按请求计费和配置单次请求价格。
5. 更新模型展示侧价格文案，避免按次模型显示 token 单价。
6. 新增 SQL 升级脚本并同步初始化 SQL。
7. 运行后端编译和前端 lint 验证。
