# 渠道联调实施方案

## 用户原始需求

连接正式环境 MySQL，只读取 `ai_model_proxy` 中已接入的 3 个媒体渠道配置：

- `21 首尔-gpt-image-2`
- `16 ChatGPT Image Compatible`
- `38 首尔-Grok`

将生图和视频生成渠道同步到本地数据库，并基于本地渠道进行功能测试。

## 技术方案设计

- 使用正式库只读查询渠道、模型映射、统一模型和图片分辨率规则。
- 根据正式库数据更新本地 `modelinvoke` 数据库中的 `channel`、`unified_model`、`model_channel_mapping`、`model_image_resolution_rule`。
- 保持本地媒体工作台代码路径不变，测试仍走 `/v1/images/generations`、`/v1/image/edit`、`/v1/created/video`。
- 测试重点：
  - `1K` 生图能路由到 compatible 优先渠道。
  - `2K/4K` 生图能路由到 native size 渠道。
  - 多参考图图片编辑能按重复 `image` 字段进入代理。
  - 视频图生视频能按重复 `input_reference[]` 字段进入代理。

## 涉及文件清单

- 本地数据库：`channel`、`unified_model`、`model_channel_mapping`、`model_image_resolution_rule`
- 验证文档：`md/impl-渠道联调-20260620.md`

## 实施步骤概要

1. 查询正式库目标渠道和映射数据。
2. 查询本地库结构和当前目标模型状态。
3. 生成并执行本地 upsert SQL。
4. 运行后端服务或直接调用本地代理接口测试。
5. 记录测试结果与问题。
