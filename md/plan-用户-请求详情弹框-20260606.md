# 用户-请求详情弹框 Plan 20260606

## 用户原始需求

当前 `user/balance` 页面对于用户的请求，可以点击成功或失败状态弹框显示请求详情。参考 `admin/logs` 页面的请求详情弹框，但用户页面只显示请求的模型 ID，不显示实际模型、渠道等内部信息；主要让客户能看到请求失败的错误详情信息。

## 技术方案设计

- 复用 `frontend/src/views/user/BalanceLog.vue` 当前请求日志数据源 `/api/user/profile/usage-logs`。
- 用户接口当前已返回 `error_message`，但需在返回给用户前做白名单 DTO 与错误详情脱敏，避免历史或流式失败日志中的实际模型、渠道等内部信息随错误文本暴露。
- 将状态列点击行为从展开计费行调整为打开详情弹框；表格行点击仍保留展开计费行。
- 弹框参考 `frontend/src/views/admin/RequestLog.vue` 的视觉结构，但仅展示用户可见字段：
  - 请求状态、请求 ID、请求模型 ID、请求类型、请求时间、响应耗时
  - Token/图片积分用量概览、计费概览
  - 错误详情与复制按钮
- 明确不展示：实际模型、渠道名称、用户信息、客户端 IP、内部路由信息。

1. 在用户账单页面增加详情弹框模板。
2. 增加 `selectedRecord`、`detailModalVisible` 状态和弹框标题计算逻辑。
3. 将状态点击改为打开详情弹框，保留行点击展开计费明细。
4. 增加错误详情复制方法与弹框容器方法。
5. 为用户日志接口增加白名单 DTO、错误详情脱敏和状态过滤参数。
6. 补充用户页弹框样式，复用当前页面视觉语言。
7. 运行前端构建与后端语法检查验证。
