# 用户端页面性能优化 Plan

## 用户原始需求

深度分析当前目录下的项目及相关代码。当前用户端页面存在大量 CSS、JS 样式特效和 UI 动效，对电脑性能造成明显影响。需要先优化用户端各个页面的 UI、UX，在尽可能保证页面样式效果的情况下，简化 CSS/JS 代码并减少性能影响。

## 任务级别

大型任务。

原因：优化范围覆盖 `frontend/src/App.vue` 全局背景、`frontend/src/layout/UserLayout.vue` 用户端布局、`frontend/src/views/user/*` 多个用户页面，以及 `frontend/src/views/common/*`、`frontend/src/components/chat/*` 公共聊天组件。涉及持续动画、滤镜、列表过渡、图表 resize、计数动画、页面样式策略等多类性能点，步骤数超过 10。

## 当前项目分析

### 技术栈

- 前端：Vue 2.7、Vue Router 3、Vuex 3、Ant Design Vue 1.7、Less。
- 构建：Vue CLI 5。
- 可视化：ECharts 5，主要用于用户端用量统计。
- 动画：当前通过全局和页面级 `@import animate.css` 引入 Animate.css，并大量使用 `animate__animated` 类。

### 高成本来源

1. 全局背景 `App.vue`
   - 固定定位大面积 `filter: blur(100px)`。
   - 3 个超大 blob 持续动画。
   - 所有页面都会承受背景合成和重绘成本。

2. Animate.css 重复引入
   - `App.vue` 已全局引入 Animate.css。
   - 多个用户页面又重复 `@import animate.css`。
   - 页面模板大量使用 `animate__fadeIn*` 和 `animation-delay`，首屏会同时触发多组动画。

3. 玻璃拟态和大阴影
   - 多处 `backdrop-filter: blur(10px-20px)`、大半透明卡片和大面积 `box-shadow`。
   - 对滚动容器、卡片列表、表格和弹窗影响明显。

4. 列表过渡和 hover 变换
   - `ModelList.vue` 使用 `transition-group` 渲染模型卡片列表，筛选时产生批量入离场和 move 动画。
   - 多处使用 `transition: all`，浏览器需要追踪过多属性变化。

5. 持续动画
   - `UserLayout.vue` logo 呼吸动画。
   - `Recharge.vue` hero blob、扫码线、状态 pulse。
   - `UsageRanking.vue` 榜首悬浮旋转。
   - `ChatMessage.vue` 图像生成/加载状态粒子、扩散、进度条等循环动画。

6. 图表和事件
   - `UsageStats.vue` 监听 window resize 后直接 resize 三个 ECharts 实例，缺少节流。
   - 统计页在首屏有多个 CountTo 和图表初始化，动画与图表渲染叠加。

## 技术方案设计

### 总体策略

保留产品的科技感、层次感、品牌色和关键状态反馈，但把“持续、全局、大面积”的视觉效果降级为“静态、局部、按需”。优先处理性能收益高且风险低的样式与交互：

- 大面积 blur 改为半透明实色、轻微渐变或小范围阴影。
- `transition: all` 改为限定 `background-color`、`border-color`、`color`、`transform` 等必要属性。
- 去除重复 Animate.css import，首屏动画使用轻量 CSS 或直接静态呈现。
- 移除列表 `transition-group`，筛选结果直接 grid 渲染。
- 循环动画仅保留关键支付状态提示，其余改为静态高亮。
- 支持 `prefers-reduced-motion`，系统偏好减少动画时统一关闭过渡/动画。
- 对 resize、滚动等高频事件增加节流或只在必要页面启用。

### 优化分层

1. 全局层
   - `App.vue` 移除远程 Animate.css import，改为本地少量兼容类，避免每页重复远程样式依赖。
   - 全局 aurora 从持续移动的大 blur blob 改为静态径向渐变背景。
   - 新增 `prefers-reduced-motion` 全局规则。

2. 用户布局层
   - `UserLayout.vue` 去除 logo 呼吸 keyframes 和过度阴影。
   - 侧边栏、菜单、头部保留深色/磨砂风格，但减少 blur 和 `transition: all`。
   - 主内容滚动区域保持结构不变。

3. 用户页面层
   - 对所有 `frontend/src/views/user/*.vue` 删除重复 Animate.css import。
   - 保留现有 class 名，不大规模改模板，降低回归风险。
   - 优先替换高成本样式：
     - `backdrop-filter` 改成半透明实色或轻量背景。
     - 大阴影降低模糊半径和透明度。
     - hover 位移幅度缩小。
     - 状态点 glow 降低或移除。

4. 模型列表页
   - `ModelList.vue` 将 `transition-group` 改为普通 `div.model-grid`。
   - Provider 颜色、图标、价格文案等可在后续迭代预计算；本轮优先移除列表批量动画和昂贵卡片效果。

5. 用量统计页
   - `UsageStats.vue` resize 处理增加 `requestAnimationFrame` 或 timeout 节流。
   - 销毁阶段清理待执行 resize 任务。
   - CountTo 动画时长可缩短，图表卡片减少 blur 和 hover 位移。

6. 充值页
   - `Recharge.vue` hero 背景 blob 改为静态渐变。
   - 支付二维码扫码线动画改为静态提示线或仅保留轻量状态点。
   - 状态 pulse 降低为静态绿色点。

7. 聊天公共组件
   - `ChatMessage.vue` 移除或降级粒子扩散、核心发光、环形扩散等循环动画。
   - `AiChat.vue`、`MobileChat.vue`、`ModelSelector.vue`、`SessionList.vue` 降低 glass blur、`transition: all` 和大阴影。
   - 保留消息可读性、操作按钮、滚动到底部等关键 UX。

### 风险点

- 去除 Animate.css import 后，如果仍依赖 `animate__animated` 类，需要提供兼容基础类，避免 class 失效导致样式异常。
- 降低 `backdrop-filter` 后玻璃质感减弱，需要通过边框、背景透明度和阴影维持层次。
- `ModelList.vue` 去除 `transition-group` 后筛选切换不再有入场动画，但交互会更稳定。
- `UsageStats.vue` 图表 resize 节流需要确保最终尺寸仍正确。
- 聊天组件如果直接删除过多加载动画，可能降低状态反馈，需要保留最小可感知反馈。

## Plan v2 修订

### 量化验收标准

1. 构建验证：`cd frontend && npm run build` 必须通过。
2. 静态扫描：
   - `frontend/src` 范围内远程 `animate.css` import 清零。
   - 用户端与聊天相关范围内 `transition: all`、`backdrop-filter`、`animation: ... infinite` 数量显著下降。
   - `ModelList.vue` 不再使用 `transition-group` 渲染模型卡片列表。
3. 运行回归：
   - 用户端：`/user/dashboard`、`/user/models`、`/user/stats`、`/user/recharge`、`/user/chat`、`/user/m-chat`。
   - 共享组件影响入口：`/admin/chat`、`/admin/profile`、`/admin/ranking` 如环境可访问则抽样检查。
4. 性能观察：
   - 滚动页面时不应出现明显持续动画占用。
   - 模型列表筛选时不应触发批量入离场动画。
   - 统计页窗口 resize 不应连续触发多次 ECharts resize。

### 样式降级边界

- `backdrop-filter`：默认移除；仅弹窗、顶栏等少量浮层允许保留小半径 blur，优先使用半透明实色、浅边框和低成本渐变。
- `box-shadow`：卡片阴影控制在低透明度、小模糊半径，避免大面积 30px-50px 阴影。
- `hover`：位移控制在 `translateY(-2px)` 或改为背景、边框、颜色变化。
- `transition`：禁止新增 `transition: all`，已有场景改为明确属性。
- 循环动画：只保留支付、加载等业务反馈，且受 `prefers-reduced-motion` 统一限制。
- 兼容动效：保留 `animate__*` class 名，提供本地轻量兼容类，避免大范围模板改动。

### 共享组件影响范围

以下文件虽然服务用户端优化，但会被管理端或其他入口复用，实施和回归时按共享组件处理：

- `frontend/src/views/common/AiChat.vue`：`/user/chat` 与 `/admin/chat` 共用。
- `frontend/src/views/user/Profile.vue`：`/user/profile` 与 `/admin/profile` 共用。
- `frontend/src/views/user/UsageRanking.vue`：`/user/ranking` 与 `/admin/ranking` 共用。
- `frontend/src/components/chat/*.vue`：桌面聊天和相关入口共用。
