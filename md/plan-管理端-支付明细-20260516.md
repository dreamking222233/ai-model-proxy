## 用户原始需求

当前管理端只能看到代理端及其用户端的充值情况，看不到直属用户的充值。

需要新增一个页面，例如“支付明细”或“充值明细”，能够看到每一笔线上支付的详情，包括但不限于：
- 支付平台，如支付宝 / 微信
- 用户名称
- 用户 ID
- 是否为直属用户还是代理客户
- 支付金额
- 订单号

## 技术方案设计

### 目标

在管理端新增独立的“支付明细”页面，统一查看平台直营用户与代理站用户的所有线上充值订单。

### 后端方案

复用现有 `payment_recharge_order` 表作为明细主数据来源。该表已经包含：
- 订单号
- 支付渠道
- 用户 ID
- 代理 ID
- 站点范围
- 支付金额
- 到账美元
- 代理分润
- 订单状态
- 支付宝交易号
- 支付时间

在管理端支付接口下新增或扩展“订单明细列表”能力，支持：
- 分页
- 按订单状态筛选
- 按支付渠道筛选
- 按站点范围筛选（platform / agent）
- 按代理 ID、用户 ID 筛选
- 按关键词筛选（订单号 / 用户名 / 代理名 / 域名）

返回结果中补充：
- 用户名
- 代理名
- 客户类型文案字段
- 支付平台文案字段

### 前端方案

在管理端新增路由 `/admin/payment-orders`，页面名称为“支付明细”。

页面包含：
- 顶部筛选区
- 订单列表表格
- 支持查看直营与代理全部线上支付记录
- 明确展示“直属用户 / 代理客户”

保持现有“代理提现”页面职责不变，避免职责混杂。
