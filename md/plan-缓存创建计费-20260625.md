# 缓存创建计费

## 用户原始需求

当前 Claude 新上游返回 `usage.cache_creation_input_tokens`，该缓存创建是计费的。系统需要：

- `admin/models` 支持配置模型的缓存写入/创建计费价格。
- 后端按缓存创建 token 计算费用并写入请求日志、消费记录。
- `admin/logs`、`admin/agent-logs`、`agent/logs`、`user/balance` 的请求计费详情展示缓存创建价格与计算过程。
- 修复缓存读取价格展示不一致问题：展示应为实际计费用的输入价格 10%，而不是综合倍率后的价格。

## 技术方案设计

- 在 `unified_model` 新增 `cache_creation_price_per_million`，默认 0，表示缓存创建每百万 token 单价。
- 在 `request_log`、`consumption_record` 新增：
  - `cache_creation_cost`
  - `cache_creation_price_per_million_snapshot`
- 计费时按 `billable_cache_creation_tokens = upstream_cache_creation_input_tokens * token_multiplier` 计算：
  - `cache_creation_cost = billable_cache_creation_tokens / 1M * cache_creation_price_per_million * effective_price_multiplier`
  - 套餐额度口径使用 raw cache creation tokens 和 official quota multiplier。
- 日志详情返回新增字段，前端按新字段展示缓存创建单价、费用和公式。
- 缓存读取展示统一使用基础单价 `input_price_per_million * 0.1`，倍率单独展示，不再把倍率提前算进“单价”。
