# 联系方式门槛实施方案

## 用户原始需求

当前直营用户登录后会弹出平台公告，公告中显示平台 QQ 和微信；管理端可通过 `admin/announcements` 页面配置。直营用户在 `user/recharge` 页面支付说明区域也能看到 QQ 和微信。现在希望不是每位用户都能看到 QQ 和微信，而是设置门槛：用户通过 `user/recharge` 页面充值余额或套餐，实际成功付款人民币超过 100 元后，才显示 QQ 和微信；未达到门槛时，在对应显示区域提示需要先完成充值门槛。代理端逻辑保持不变，只修改管理端和直营客户页面。

## 技术方案设计

1. 后端新增直营联系方式可见性判断：仅平台直营用户（`current_user.agent_id is None`）参与门槛判断，代理用户直接沿用原展示逻辑。
2. 门槛口径：统计当前用户 `payment_recharge_order` 中 `status='paid'`、`agent_id IS NULL`、`site_scope='platform'`、`recharge_type IN ('balance', 'subscription')` 的 `amount_cny` 累计值；累计金额大于管理端配置的门槛时允许显示 QQ/微信，默认门槛为 100 元。
3. `GET /api/user/profile/site-config` 对直营未达标用户返回空的 `support_wechat/support_qq`，并追加 `support_contact_visible/support_contact_threshold_cny/support_contact_paid_cny/support_contact_notice` 等字段，供充值页提示使用。代理站点保持原返回。
4. `GET /api/user/profile/announcements` 对直营固定公告联系方式做相同脱敏，未达标时返回提示文案字段，登录开屏弹窗和公告抽屉显示提示而不显示真实联系方式。额外平台公告不携带联系方式，不变。
5. `GET /api/public/site-config` 无登录用户上下文，直营站点不返回 QQ/微信，避免公开接口绕过登录后脱敏；代理公开站点保持原逻辑。
6. 管理端 `AnnouncementManage.vue` 在固定公告配置中增加联系方式展示门槛输入框，并提示 QQ/微信会对直营用户按充值门槛展示，避免管理员误解配置失效。
7. 用户端 `Dashboard.vue`、`UserLayout.vue`、`Recharge.vue` 按后端字段统一展示：可见时显示 QQ/微信，未达标时显示门槛提示。

## 实施步骤概要

1. 在用户资料接口中引入 `PaymentRechargeOrder` 与 SQL 聚合，封装联系方式门槛计算和脱敏方法。
2. 改造用户站点配置接口，对直营未达标用户隐藏联系方式并返回提示元数据。
3. 改造用户公告接口，对直营固定公告联系方式做同口径隐藏和提示。
4. 隐藏公开直营站点配置接口中的 QQ/微信，防止未登录绕过。
5. 更新管理端公告配置页，允许配置直营联系方式展示门槛。
6. 更新充值页支付说明区域，未达标时展示门槛提示，达标后展示联系方式。
7. 更新登录公告弹窗和公告抽屉联系方式区域，支持提示文案。
8. 同步初始化 SQL 默认配置。
9. 执行后端语法检查与前端构建验证，创建实施记录并进行自审。
