# plan-视频-Grok119337渠道-20260704

## 用户原始需求

当前 grok 视频生成采用 CPA Grok 视频渠道类型。根据新的 Grok Video API 文档，新增一个独立渠道类型，支持上游 `https://api.119337.xyz` 的异步视频任务创建、查询和结果下载。

## 技术方案设计

- 新增 OpenAI provider variant：`grok-video-119337`。
- 管理端渠道表单支持选择该渠道类型，并默认关闭健康检查。
- 后端视频代理在 `ProxyService` 中增加该渠道适配：
  - 创建：`POST {base_url}/v1/video/generations`
  - 查询：`GET {base_url}/v1/video/generations/{task_id}`
  - 下载：查询成功后取 `data.result_url` 并代理下载。
- 对用户侧保持现有统一接口不变：
  - `POST /v1/videos`
  - `GET /v1/videos/{video_id}`
  - `GET /v1/videos/{video_id}/content`
- 参数映射：
  - `size` 转 `aspect_ratio`
  - `resolution_name` 转 `resolution`
  - 上传参考图转为 data URL，放入 `image_urls`
  - 多参考图最多 7 张；多参考图 seconds 超过 10 时按 10；整体 seconds 支持 4/6/8/10/12/15 并做范围归一。
- 响应归一：
  - `data.status == SUCCESS && data.result_url` -> `completed`
  - `data.status == FAILURE` -> `failed`
  - `SUBMITTED/QUEUED/IN_PROGRESS/NOT_START` -> `queued/in_progress`
