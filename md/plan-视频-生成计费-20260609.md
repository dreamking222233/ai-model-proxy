# plan-视频-生成计费-20260609

## 用户原始需求

当前系统已经接入视频生成，但存在上游实际失败、未返回视频路径或未返回正确视频内容时，系统仍按成功计费的问题。需要优化视频生成处理逻辑：

- 错误信息正确处理并返回。
- 确认成功后才计费。
- 失败不计费。
- 覆盖用户调用视频生成时可能遇到的上游错误，例如 input reference 上传失败 403。

## 技术方案设计

当前视频生成存在两条链路：

1. OpenAI Chat Completions 视频链路：`/v1/chat/completions` 中 `model_type=video` 进入 `handle_video_chat_completions_request`，最终调用上游 `/v1/chat/completions`。
2. 专用视频任务链路：`/v1/videos` 创建任务，上游 `/v1/videos`；`/v1/created/video` 会创建后等待完成。

本次优先修复实际导致“无视频仍计费”的核心点：

- OpenAI Chat Completions 非流式视频分支：
  - 上游 HTTP 2xx 后先解析响应。
  - 检查顶层 `error`、message content 中的失败文本。
  - 必须提取到视频 URL。
  - 对视频 URL 做轻量校验：请求 URL，确认 HTTP 成功、响应非 JSON 错误、内容非空。
  - 校验通过后才 `_log_video_success` 扣费和写成功日志。
- OpenAI Chat Completions 流式视频分支：
  - 流结束后必须从收集文本中提取视频 URL。
  - 无 URL 或文本包含失败语义时写失败日志，不扣费。
  - URL 校验通过才计费。
- `/v1/created/video` 等待分支：
  - 创建任务目前在 `handle_video_request` 中已扣费，等待失败不会回滚。
  - 新增可选参数让等待场景延迟计费：创建任务只返回 task id 和记录路由，等待完成后再扣费。
- `/v1/videos` 纯异步分支：
  - 创建阶段只保存任务路由和计费上下文，不扣费。
  - 下载 `/content` 前校验上游内容，校验通过后幂等扣费。
