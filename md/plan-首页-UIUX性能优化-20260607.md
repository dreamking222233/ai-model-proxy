# 首页-UIUX性能优化 Plan

## 用户原始需求

优化 `frontend/src/views/Home.vue` 页面的 UI、UX。当前页面打开有点卡顿，要求在保留当前页面效果的前提下最大程度进行优化。

## 技术方案设计

1. 保留现有首页视觉结构：科技背景、Hero、3D 大模型星系、核心优势、模型列表与页脚。
2. 降低首屏和后台 CPU 占用：
   - Canvas 粒子动画按设备性能和视口尺寸自适应数量、低帧率调度，并在页面不可见时暂停。
   - 支持 `prefers-reduced-motion`，弱性能/小屏设备降级为静态或轻量动效。
   - 清理 resize、visibility、定时器等监听，避免页面切换后残留。
3. 降低 Vue 渲染成本：
   - 将模型品牌 SVG、品牌 class、价格文案、星系节点 style 预计算到模型视图数据，避免模板每次渲染重复调用。
   - 首屏先用兜底模型渲染，接口返回后再替换，减少空白等待。
   - 模型列表使用普通 grid 替代 `transition-group` 的批量入离场动画，减少列表切换重排。
4. 优化交互体验：
   - 悬停状态避免重复赋值触发不必要的响应式更新。
   - 点击定位模型卡片时管理高亮定时器，避免重复 setTimeout。
   - 移动端缩小星系半径和节点尺寸，避免横向溢出。
5. CSS 层性能优化：
   - 对高频动画元素添加合适的 `will-change`、`backface-visibility`、`contain`。
   - 降低大范围 blur/backdrop-filter 的渲染成本，同时保持当前风格。
