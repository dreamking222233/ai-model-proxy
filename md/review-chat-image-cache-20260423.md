# 聊天图片缓存与模型筛选优化 Review（2026-04-23）

## 审查结论
- 当前实现已满足本轮前端需求：
  - 聊天页图片刷新后可恢复
  - 删除会话 / 清空会话时会清理对应图片缓存
  - “查看原图”不再直接依赖 `data:` URL 新开页
  - 图片支持放大预览与下载原图
  - 图片编辑发送后会清空上传区
  - 模型选择区域支持按厂商、类型和关键词筛选

## 本轮关键修正
1. 图片从运行时内存改为 IndexedDB 持久缓存
- 解决刷新后图片消失问题。
- 同时避免把大图 base64 挤进 localStorage。

2. 原图查看方式改为 blob 打开
- 规避浏览器对 `data:` URL 新标签页的拦截。

3. 修图上传区发送后主动清空
- 提升连续修图场景的输入体验。

4. 模型选择器改为筛选面板
- 解决模型变多后单纯搜索下拉框不好选的问题。

## 残余风险
1. 浏览器 IndexedDB 可用性依赖客户端环境
- 主流现代浏览器通常可用；若用户浏览器禁用了 IndexedDB，则图片持久缓存会退化失效。

2. 图片缓存生命周期仍由浏览器管理
- 若用户主动清理站点数据或浏览器自动回收存储，缓存图片仍可能消失。

3. 当前图片缓存未做容量上限控制
- 本轮按“清空会话即清缓存”处理；如果单用户连续生成大量图片，后续可再补容量策略。
