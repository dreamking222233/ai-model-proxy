# Chat 单图编辑回归修复 Review（2026-04-23）

## 结论
- 本轮修复符合需求，未发现阻断问题。
- 单图编辑卡在“生成中”的根因已定位并修复：发送请求时引用了已经被 UI 清空的上传文件状态。

## 核查结果

### 1. 根因是否准确
- 是。
- `handleEditImage()` 在请求前执行 `clearEditImage()`，而 `sendEditImageRequest()` 又依赖 `this.editImageFile` 组装 `FormData`。
- 这是一个典型的前端状态时序问题，与后端多图能力本身无关。

### 2. 修复方式是否合理
- 是。
- 使用发送快照保留文件与文件名，允许 UI 及时清空，同时保证实际请求使用稳定参数。
- 该方式改动范围小，不影响图片生成与编辑结果展示逻辑。

### 3. 风险评估
- 当前修复聚焦单图编辑发送链路，不影响：
  - 图片生成
  - 后端多图编辑透传
  - 图片缓存与预览逻辑
- 后续若前端要支持聊天页多图上传，可沿用“发送快照”模式扩展到文件数组。

## 非阻断建议
- 后续可在前端补一个更明确的提交态提示，例如编辑请求开始后禁用重复提交按钮，避免用户误判为未发送。
