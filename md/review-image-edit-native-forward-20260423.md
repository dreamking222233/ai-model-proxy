# 图片编辑原生透传修复 Review（2026-04-23）

## 结论
- 本轮修复符合需求，未发现阻断问题。
- 当前系统多图编辑与上游原生调用之间的主要差异点已收敛到最小。

## 核查结果

### 1. 多图编辑是否仍然完整透传
- 是。
- 图片编辑仍通过重复 `image` multipart 字段转发上游。
- 现有测试已断言第一张图、第二张图都会进入上游请求。

### 2. 编辑 prompt 是否仍被系统改写
- 否。
- 当前 `ProxyService._build_openai_image_edit_prompt()` 已改为直接返回原始 `prompt`。
- 不再附加 `image_size`、`aspect_ratio` 和“保留原图主体”等增强文本。

### 3. 风险评估
- 本次调整只影响图片编辑场景，不影响文生图 prompt 适配逻辑。
- 已保留：
  - `n=1`
  - `response_format=b64_json`
  - 多图上传
  - 计费、日志和超时逻辑

## 非阻断建议
- 如果后续仍需要在少数模型上做“编辑提示增强”，建议改成显式开关，而不要默认对所有图片编辑请求进行 prompt 改写。
- 若要进一步做线上比对，可直接用同一组图片和 prompt 对比：
  - 直连上游 `/v1/images/edits`
  - 经系统转发 `/v1/image/edit`
  核对最终请求体中的 `prompt` 是否一致。
