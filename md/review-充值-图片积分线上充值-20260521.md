# 充值-图片积分线上充值 Review

## 审查结论

通过。实现覆盖用户图片积分线上充值、代理现金分润、管理端/代理端展示与筛选、数据库升级和初始化脚本同步。

## 审查项

- 支付订单兼容性：`recharge_type` 默认 `balance`，历史订单按余额订单处理，`credited_image_credits` 默认 0。
- 图片积分到账：支付成功后写入 `user_image_balance` 和 `image_credit_record`，不写入代理图片积分池，符合“用户充值到账本人，代理只拿现金分润”的需求。
- 代理分润：图片积分订单按用户零售价 `1 元 = 5 积分`、代理拿货价 `1 元 = 10 积分` 计算，1 元订单分润 0.5 元；平台直营订单分润为 0。
- 幂等性：仍复用原订单锁与状态判断，支付回调/查单重复触发不会重复入账。
- 展示与筛选：管理端支付订单、管理端代理现金、代理端支付页均已展示充值类型和余额/图片分项数据；代理端订单补充了充值类型筛选。
- SQL 一致性：新增增量升级脚本，同时同步 `backend/sql/upgrade_alipay_recharge_20260516.sql` 和 `sql/initData.sql`，避免新环境初始化缺字段。

## 发现与修复

- 发现 `sql/initData.sql` 和旧在线充值建表脚本缺少新字段，已补充 `recharge_type`、`credited_image_credits` 和 `idx_payment_recharge_type`。
- 发现代理端支付页只展示充值类型但不能筛选，已补充充值类型筛选并传递给后端接口。
