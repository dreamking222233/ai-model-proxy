# 套餐每日限额预检漏洞代码审查

## 首轮审查

发现 2 个问题：

1. 余额兜底记录仍关联 `subscription_id`，可能被套餐用量统计误算。
   - 处理结果：已在余额兜底分支清空 `subscription_id`，并在套餐用量汇总/周期重建中过滤 `billing_mode="balance"` 记录。
2. 原子限额更新缺少有效测试，旧 mock 用例仍模拟套餐可超额记账。
   - 处理结果：已将测试改为覆盖 `_apply_cycle_consumption_update` 返回 `False` 时抛出 `SUBSCRIPTION_QUOTA_UPDATE_FAILED`，不再断言超额写入。

## 复审结论

结论：通过。

问题清单：未发现阻断问题。

复审重点：

1. 套餐额度不足但余额足够时，实际记账改走余额扣费，并清空套餐关联字段。
2. 余额兜底记录不会计入套餐用量汇总或周期重建。
3. 套餐额度和余额都不足时返回中文错误：`当日套餐额度不足，且余额不足以承担本次请求，请充值或缩短上下文后重试`。
4. 套餐周期更新使用 `used_amount + consumed_amount <= quota_limit` 原子条件，未发现每日限额继续写穿路径。
