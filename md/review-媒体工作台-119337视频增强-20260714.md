### 已修复：旧 default Grok 渠道秒数能力不准确

首次审查发现，能力兜底直接复用了 `VIDEO_SECONDS_CAPABILITIES` 中 Grok 的 `1-15`，但旧 default OpenAI-compatible 视频链路实际只接受 `6/10/12/16/20`，会让页面展示无法提交的值。

修复：

- `grok-video-119337`：`4/6/8/10/12/15`，有参考图 `4/6/8/10`。
- `cpa-grok-video`：`1-15`。
- `default`：`6/10/12/16/20`。
- `zz1cc-video`：固定 `15`。

新增测试验证 default 与 CPA 的秒数能力分离。
