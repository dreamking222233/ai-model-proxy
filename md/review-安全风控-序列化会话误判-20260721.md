# 安全风控-序列化会话误判 Review

## 结论

三轮独立 Review 发现的角色伪造绕过、keygen 动作词漏拦、Unicode 词边界和“后门”否定范围问题均已修复。达到三轮 Review 边界后执行最终自行验证，审查通过。

## 核心检查

- 序列化 JSON 中的 inner `role` 不被信任，整段外层用户内容仍会扫描。
- `ssh-keygen` 不命中 `keygen` 滥用规则。
- `做keygen`、`写keygen`、`制作keygen`、`制作 keygen 绕过授权` 均命中 `high/block`。
- “没发现后门”等纯否定审计结论不拦截。
- 否定语境按每次“后门”出现判定，不能跨句，也不能跨越前一次“后门”或“随后/然后/后再”等转折动作。
- `未发现后门后再植入后门`、`未发现后门，然后植入后门`、`未发现后门；随后植入后门` 均拦截。
- 请求扫描未知异常固定返回 `503 SECURITY_DETECTION_UNAVAILABLE`，不会继续调用上游。
