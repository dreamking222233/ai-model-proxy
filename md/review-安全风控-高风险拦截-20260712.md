# 安全风控-高风险拦截 Review

## 结论

审查通过。

## 核心检查

- 请求前 `keyword` 扫描判定为 `high` 或 `blocked` 时，后端强制规范为 `action=block`，记录风控事件并返回 `403 SECURITY_RISK_BLOCKED`。
- 高风险拦截不再受 `security_keyword_block_enabled` 开关影响。
- 拦截文案查询异常时使用内置文案，风控事件落库失败时重试并写 critical 备用审计日志，均不会放行请求。
- 请求扫描未知异常统一 fail-closed，返回 `503 SECURITY_DETECTION_UNAVAILABLE`；扫描调用位于渠道选择和上游请求之前。
- `medium/review` 请求只记录不拦截。
- `model_report` 和 `model_report_api` 是上游响应后的模型自报/事后上报，不是请求前判级，保留 `action=review` 才能准确表示“仅记录”。
- 前端正确展示 `block -> 已拦截`、`review -> 仅记录`。
