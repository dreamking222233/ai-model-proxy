# 推广返利配置 Review

## 结论

实现符合需求：推广返利比例已从固定 20% 改为系统配置 `promotion_reward_rate`，可在 `admin/config` 页面按百分比修改。配置为 `0` 时不会创建推广返现记录，也不会给推广人增加余额或图片积分。

## 已检查事项

- 后端余额返现和图片积分返现均使用同一动态比例。
- 配置缺失、空值或非法值时默认保持原 20% 行为。
- 配置范围限制为 0 到 1，避免负数或超过 100%。
- 前端展示为百分比，保存为小数，符合后端读取格式。
- 升级 SQL 使用 `ON DUPLICATE KEY UPDATE`，可重复执行。

## 备注

支付入账流程只在 `PaymentService._apply_paid_order` 的新结算分支调用推广返利，因此重复支付回调不会因为日后恢复比例而给同一订单补发返利。
