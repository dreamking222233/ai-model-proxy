**Findings**

No blocking issues found for the two重点复核项。

- 非法 `reasoning.effort` 已修复：`_has_explicit_responses_reasoning_effort()` 现在只有在值属于 `minimal/low/medium/high/xhigh` 时才认为用户显式设置；非法值不会再阻止映射默认值生效。已用函数级脚本验证 `ultra -> xhigh`、合法 `low` 保留。
- 隐式 Responses bridge 已修复：`default_reasoning_effort` 已传入 `_stream/_non_stream_anthropic_via_responses_request()`，并在 `_convert_anthropic_request_to_responses()` 内写入 `responses_request["reasoning"]`，覆盖了基于 `base_url` 的隐式 bridge 场景。

**Residual Risk**

OpenAI 官方文档显示 `xhigh` 是合法枚举，但不是所有模型都支持，例如 `gpt-5.1` 不支持 `xhigh`，`gpt-5-pro` 只支持 `high`。当前实现只做全局枚举校验，不做按上游模型的兼容性校验；如果管理员给不支持的模型配置 `xhigh`，仍可能由上游返回 400。这个不违背当前 plan，因为 plan 明确允许 `xhigh`，但属于后续可增强点。来源：OpenAI Responses API Reference, `reasoning.effort`。
