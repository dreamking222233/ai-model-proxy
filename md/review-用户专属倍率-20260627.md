# 用户专属倍率 Review 记录

## Review 说明

已执行 `codex exec` 二次审查，但审查过程在末尾因上游 `429 Too Many Requests` 中断，未生成完整最终结论。中断前已检查到：

- 用户专属倍率规则 CRUD 存在用户维度，并校验 `rule.user_id == path user_id`。
- `PriceAdjustmentService.resolve_adjustment()` 采用 `用户专属 > 全局规则 > 默认 x1`，用户专属规则不是和全局规则叠乘。
- OpenAI、Anthropic、Responses 三类文本入口均构建了冻结倍率上下文。
- SQL 初始化和升级脚本包含 `user_price_adjustment_rule`、倍率快照字段、`video_task_billing_snapshot`。
- 缺口：当时尚未新增自动化测试。

## 人工复核与修复

基于中断前审查输出和人工复核，已继续修复：

- 修复 Responses 安全检查失败分支引用未初始化 `request_billing_context` 的运行时风险。
- 预检失败日志补充 `billing_context`，失败请求也能在 `admin/logs` 展示请求时倍率。
- 调整 `admin/users` 路由顺序，确保 `/{user_id}/price-adjustments...` 优先于 `/{user_id}`。
- 新增 `backend/test_user_price_adjustment.py`，覆盖用户专属覆盖全局、冻结上下文、视频直链净化、媒体积分用户倍率传参。

## 审查结论

当前实现符合需求：

- 可以在 `admin/users` 为单个用户配置某个模型系列/类型/计费类型倍率。
- 用户专属倍率命中后覆盖全局分类倍率，未命中回退全局，再回退默认 x1。
- `admin/price-adjustments` 可查看用户专属规则列表。
- `admin/logs` 已在“响应时间”和“计费”之间展示“用户倍率”列。
- 文本预检与最终扣费使用同一倍率快照。
- 异步视频创建时持久化倍率快照，查询 completed 时会扣费，返回体会净化上游直连，避免绕过 `/content` 漏扣费。
