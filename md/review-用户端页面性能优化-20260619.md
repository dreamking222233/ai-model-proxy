**审查结论**

发现 4 个需要处理的问题：

1. **Reduced motion 仍会延迟隐藏内容**  
   [frontend/src/App.vue](/Volumes/project/modelInvocationSystem/frontend/src/App.vue:105) 只缩短了 `animation-duration`，没有清掉 `animation-delay`。而页面里大量元素仍有 `animation-delay`，例如 [UsageStats.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/UsageStats.vue:56)。由于 `.animate__animated` 使用 `animation-fill-mode: both`，开启减少动态效果的用户仍可能看到内容延迟出现。建议在 reduced-motion 中加：
   ```css
   animation-delay: 0ms !important;
   animation: none !important;
   ```

2. **多处 hover 位移和大阴影不符合 Plan v2 降级边界**  
   Plan 写明 hover 位移控制在 `translateY(-2px)`，但仍有 `translateY(-8px)`、`-5px`、`-4px` 以及较大阴影，例如 [Dashboard.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/Dashboard.vue:656)、[ApiKeyManage.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/ApiKeyManage.vue:769)、[BalanceLog.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/BalanceLog.vue:981)。这和实施记录“高成本样式已降级”的结论不完全一致。建议统一降到 `-2px` 或改为边框/背景变化。

3. **统计页 JS 动画仍偏重，未纳入 reduced-motion**  
   `count-to` 仍是 1600ms：[UsageStats.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/UsageStats.vue:42)、[Dashboard.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/Dashboard.vue:67)。ECharts 也未显式降低/关闭动画：[UsageStats.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/UsageStats.vue:292)。CSS reduced-motion 不会影响这些 JS 动画。建议加 `prefers-reduced-motion` 检测，减少或关闭 CountTo/ECharts 动画。

4. **UsageStats resize 仍可能每帧触发图表重绘**  
   [UsageStats.vue](/Volumes/project/modelInvocationSystem/frontend/src/views/user/UsageStats.vue:465) 用 RAF 合并同一帧事件，但拖拽窗口时仍可能每帧执行一次，并且还会 `renderPieChart()` 重新 `setOption`。这比直接 resize 好，但不满足 Plan 中“窗口 resize 不应连续触发多次 ECharts resize”的验收口径。建议改为 100-150ms debounce，或仅在跨过 `1100px` 断点时重算饼图配置，其余只 `resize()`。

静态扫描目标项基本符合实施记录：目标范围内未发现远程 Animate.css、`transition-group`、`backdrop-filter`、`transition: all`、`animation: ... infinite`、`filter: blur(...)` 残留。`git diff --check` 通过。总体方向符合需求，但建议先修上面 4 点再认为 Review 通过。
