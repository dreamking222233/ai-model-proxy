# 代理端结算系统需求实施总结

更新时间：2026-04-30  
功能范围：代理端授信发放、销售记账、管理端结算、每日额度控制

## 一、需求背景

原代理端模式要求管理端先给代理充值余额、图片积分或套餐库存，代理才能给自己的下游用户分配资源。

这种模式的问题：

- 管理端需要频繁为代理预充值资源，运营效率低。
- 代理销售后无法形成清晰的待结算账目。
- 管理端无法按天查看每个代理实际销售情况。
- 无法支持“代理先卖，管理端按天或按周期结算”的业务模式。

本次新增代理结算系统，目标是：

- 代理可以直接给自己的用户充值余额、图片积分、开通套餐。
- 如果代理旧资产池不足，系统可以使用管理端配置的每日授信额度继续发放。
- 授信发放会自动生成待结算销售记录。
- 管理端可以查看每个代理每天的销售汇总和明细。
- 管理端可以按数量部分结算，例如卖出 5 张月卡，只结算 4 张，剩余 1 张继续未结算。
- 管理端可以限制每个代理每天可用的授信额度，防止代理滥用。

## 二、核心业务规则

### 1. 旧资产池优先

代理已有的余额、图片积分、套餐库存继续保留。

发放资源时：

- 如果旧资产池足够，优先扣旧资产池。
- 旧资产池发放不生成待结算记录，因为这部分可以理解为已经预充值、已结算资源。
- 如果旧资产池不足，则整笔走每日授信额度。

当前不做混合拆分。

示例：

- 代理余额池剩余 3 美元。
- 代理给用户充值 5 美元。
- 系统不会扣 3 美元余额池再授信 2 美元。
- 系统会整笔 5 美元尝试走授信额度。

### 2. 每日授信额度

管理端可以给每个代理配置每日可支配额度：

- 余额额度：按美元金额控制。
- 图片积分额度：按积分数量控制。
- 套餐额度：按套餐模板分别控制，例如无限月卡、无限周卡、畅享月卡等。

每日额度按北京时间自然日刷新。

规则：

- 未配置额度：不可授信。
- 额度为 0：不可授信。
- 额度停用：不可授信。
- 当日已用额度超过配置额度：不可继续授信。
- 套餐类额度必须是整数。

### 3. 待结算销售记录

当代理使用授信额度发放资源时，系统会生成销售记录。

记录内容包括：

- 代理 ID。
- 终端用户 ID。
- 北京时间业务日期。
- 资源类型。
- 套餐快照信息。
- 销售数量。
- 已结算数量。
- 当前结算状态。

结算状态：

- `pending`：未结算。
- `partial`：部分结算。
- `settled`：已结算。
- `cancelled`：预留状态，当前未开放。

### 4. 部分结算

管理端可以按代理、资源类型、套餐、日期筛选销售记录并执行结算。

结算逻辑：

- 按最早销售记录优先结算。
- 结算数量不能超过当前未结算数量。
- 套餐结算数量必须是整数。
- 每次结算都会生成结算批次和批次明细，保留审计记录。

示例：

- A 代理昨天卖出 2 张无限月卡。
- A 代理今天卖出 3 张无限月卡。
- 管理端本次只结算 4 张。
- 系统会按时间顺序划掉 4 张，剩余 1 张继续未结算。

### 5. 回收与扣减规则

为避免授信资源被扣回后变成旧资产池，当前规则如下：

- 如果代理存在未结算授信余额记录，则代理从用户扣减余额时，不回流到代理余额池。
- 如果代理存在未结算授信图片积分记录，则代理从用户扣减图片积分时，不回流到代理图片积分池。
- 已产生的待结算记录不会因为普通扣减自动冲销。
- 后续如果需要退款冲销，可新增负向结算或 `cancelled` 流程。

### 6. 兑换码边界

本次没有把代理兑换码纳入授信结算系统。

当前代理兑换码仍然使用代理余额池：

- 生成兑换码时扣减代理余额。
- 未使用兑换码删除后退回代理余额。
- 不走每日授信额度。
- 不生成结算记录。

## 四、最终实现总结

当前代理结算系统已经实现以下能力：

- 管理端可为代理配置余额、图片积分、每个套餐模板的每日授信额度。
- 代理旧资产池足够时继续走旧逻辑。
- 代理旧资产池不足时走授信额度并生成待结算销售记录。
- 代理端套餐页可看到库存和今日额度。
- 代理工作台可看到今日授信额度和待结算摘要。
- 管理端可查看代理结算汇总。
- 管理端可查看代理销售明细。
- 管理端可按数量执行部分结算。
- 管理端可看到旧资产池剩余和已消耗情况，避免和待结算销售混淆。
- 参数校验错误返回已尽量中文化。

- 余额和图片积分允许小数。

### 3. 结算服务

文件：

- `backend/app/services/agent_settlement_service.py`

新增能力：

- 查询代理每日授信额度。
- 批量保存代理每日授信额度。
- 校验并消耗当日授信额度。
- 创建待结算销售记录。
- 查询管理端结算明细。
- 查询管理端结算汇总。
- 执行部分结算。
- 判断代理是否存在未结算授信记录。

关键处理：

- 使用北京时间 `Asia/Shanghai` 计算业务日期。
- 发放时先锁额度配置行，再锁或创建使用量行。
- 批量保存每日额度时先整批校验，再统一提交，避免半成功。
- 结算时按 `created_at asc, id asc` 顺序划掉最早记录。

### 4. 代理资产服务

文件：

- `backend/app/services/agent_asset_service.py`

改造内容：

- `grant_user_balance` 支持旧余额池不足时走授信额度。

- `grant_user_subscription` 支持旧套餐库存不足时走授信额度。

## 八、部署说明

上线前需要执行 SQL：

```sql
source sql/upgrade_agent_settlement_system_20260430.sql;
```

或使用后端目录同步文件：

```sql
source backend/sql/upgrade_agent_settlement_system_20260430.sql;
```

部署后操作流程：

1. 登录管理端。
2. 打开 `admin/agent-assets`。
3. 选择代理。
4. 配置余额、图片积分、各套餐模板的每日授信额度。
5. 代理端给用户充值或开通套餐。
6. 打开 `admin/agent-settlements` 查看销售汇总和明细。
7. 根据实际收款情况执行部分或全部结算。
