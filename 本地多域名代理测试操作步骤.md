# 本地多域名代理测试操作步骤

## 一、目标

在本地模拟：

- 平台前台
- 代理前台
- 统一 API

并验证：

1. 白牌站点配置
2. 代理登录与后台访问
3. 代理用户注册归属
4. 代理日志归属
5. API Key 直连统一 API

---

## 二、推荐本地域名

### 方案 A：使用 `*.localhost`

推荐优先使用：

- 平台前台：`platform.localhost:8080`
- 代理前台：`happy.localhost:8080`
- 统一 API：`api.localhost:8085`

### 方案 B：使用 hosts

如果你本机对 `*.localhost` 使用不方便，可以加 hosts：

```txt
127.0.0.1 platform.local
127.0.0.1 happy.local
127.0.0.1 api.local
```

然后用这些域名访问。

---

## 三、已做的代码支持

### 1. 前端

`frontend/vue.config.js` 当前已经支持：

- `host: 0.0.0.0`
- `allowedHosts: 'all'`
- `/api` 和 `/v1` 代理到 `127.0.0.1:8085`
- `changeOrigin: false`

### 2. 前端请求层

`frontend/src/api/request.js` 已经支持：

- 生产环境固定共享 API 域名
- 浏览器请求自动携带 `X-Site-Host`

### 3. 后端站点识别

后端当前按以下优先级识别站点：

1. `X-Site-Host`
2. `Origin`
3. `Referer`
4. `Host`

所以本地多前台域名测试已经可以贴近正式结构。

---

## 四、本地启动方式

### 1. 启动后端

在项目根目录执行：

```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8085 --reload
```

### 2. 启动前端

在项目根目录执行：

```bash
cd frontend
npm run serve
```

## 五、数据库里如何配置测试代理

假设你要测试代理：

- 代理编码：`happy-ai`
- 代理名称：`快乐AI`

推荐配置如下。

### 使用 `*.localhost`

- `agent_code = happy-ai`
- `agent_name = 快乐AI`
- `frontend_domain = happy.localhost`
- `api_domain = NULL`
- `site_title = 快乐AI`
- `quickstart_api_base_url = http://api.localhost:8085`

### 使用 hosts

- `agent_code = happy-ai`
- `agent_name = 快乐AI`
- `frontend_domain = happy.local`
- `api_domain = NULL`
- `site_title = 快乐AI`
- `quickstart_api_base_url = http://api.local:8085`

## 六、访问方式说明

### 1. 平台管理端

访问：

- `http://platform.localhost:8080/login`

登录后：

- 管理员进入 `/admin/dashboard`

### 2. 代理前台

访问：

- `http://happy.localhost:8080/login`

登录后：

- `agent` 账号进入 `/agent/dashboard`
- `user` 账号进入 `/user/dashboard`

### 3. 统一 API

如果你要直接测试 API Key：

- `http://api.localhost:8085`

例如：

- `http://api.localhost:8085/api/public/site-config`
- `http://api.localhost:8085/v1/chat/completions`

如果你要模拟代理前台来源，记得带：

- `X-Site-Host: happy.localhost`

## 七、推荐测试顺序

### 第一步：确认平台站点

打开：

- `http://platform.localhost:8080/login`

确认：

- 能访问登录页
- 平台名称正常

### 第二步：创建代理

平台管理员登录后：

- 打开 `/admin/agents`
- 新增代理
- 填写 `frontend_domain = happy.localhost`
- `api_domain` 留空

### 第三步：给代理充值

进入：

- `/admin/agent-assets`

给该代理充值：

- 余额
- 图片积分
- 套餐库存
- 固定面额规则

### 第四步：准备代理账号

准备一个：

- `role=agent`
- `agent_id=该代理ID`

的账号，然后访问：

- `http://happy.localhost:8080/login`

### 第五步：代理端联调

验证：

1. 是否跳转到 `/agent/dashboard`
2. 是否能看到自己的站点名称
3. 是否能管理代理用户
4. 是否能给用户充余额 / 扣余额
5. 是否能生成固定面额兑换码
6. 是否能发放套餐

### 第六步：终端用户联调

终端用户从代理站点注册 / 登录后验证：

1. 是否自动绑定到代理
2. 是否读取到代理自己的公告与配置
3. 请求日志是否归属到正确代理

### 第七步：API Key 联调

使用代理用户生成的 API Key 直接访问：

- `http://api.localhost:8085/v1/chat/completions`

确认：

1. 请求能通过
2. 不需要 `happy-api.localhost`
3. 日志里的 `agent_id` 正确

---

## 八、排查重点

如果代理识别不生效，优先检查：

1. 数据库里的 `frontend_domain` 是否和浏览器访问域名完全一致
2. `quickstart_api_base_url` 是否已改成统一 API 地址
3. 前端 dev server 是否已重启
4. 后端是否已重启
5. 浏览器请求头里是否带了 `X-Site-Host`
